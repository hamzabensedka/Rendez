# Front-End Redesign

## Canonical Source

Read `docs/skills/front-end/skills.md` first. It is the project-specific source of truth for this skill.

## When To Use

Apply this skill when:

- redesigning existing screens or flows
- reducing resemblance to recognizable products
- rewriting product-specific copy, routes, or UI structure
- updating frontend UX while preserving business functionality

## Core Instructions

1. Preserve functionality, but change the user experience enough to make it clearly original.
2. Prefer neutral, industry-standard booking patterns over product-specific conventions.
3. Redesign layout, wording, interaction order, and visual system when resemblance risk exists.
4. Remove product-specific references from component names, variables, routes, and copy.
5. Prioritize clarity, hierarchy, responsiveness, and accessibility.

## Required Checks

Before finalizing:

- confirm the UI does not feel like a 1:1 reproduction
- confirm labels and actions use neutral language
- confirm repeated UI is refactored into reusable components where appropriate
- summarize how the redesign improved originality and usability
