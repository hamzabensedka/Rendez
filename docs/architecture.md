Overview\nThis document describes the architecture for the Planity Clone, a mobile app for discovering, booking, and managing appointments with local businesses. The solution uses an Nx monorepo with Expo (React Native) for the frontend, NestJS with Prisma and PostgreSQL/PostGIS for the backend, Redis for caching, and Supabase for authentication and file storage. CI/CD is handled via GitHub Actions, Docker Compose for local development, and EAS Build for Expo builds.\n\n## 2. High-Level Architecture\n- **Frontend**: Expo managed workflow, React Native, TypeScript, Expo Router for file-based navigation, TanStack React Query for data fetching, React Native Reanimated for animations, and a shared UI component library.\n- **Backend**: NestJS microservice (monolith) providing RESTful APIs, GraphQL optional, using Prisma ORM to interact with PostgreSQL (extended with PostGIS for geospatial queries). Redis used for caching and as a broker for BullMQ background jobs.\n- **Infrastructure**: Docker Compose defines services for API, database, Redis, and optionally a mock Supabase. GitHub Actions runs lint, test, and build pipelines. EAS Build creates iOS/Android binaries. Nx orchestrates monorepo tasks.\n\n## 3. Service Boundaries\n| Service | Responsibility | Technology |\n|---------|----------------|------------|\n| **frontend** | Mobile UI, navigation, state management, offline caching | Expo, React Native, TypeScript, Expo Router, React Query, Reanimated |\n| **api** | Business logic, authentication, booking flow, availability computation, notifications, admin/business portals | NestJS, Prisma, PostgreSQL + PostGIS, Redis, BullMQ |\n| **shared** | Type definitions, DTOs, validation schemas, UI constants, utility functions | TypeScript library |\n| **dev-infra** | Docker compose, CI scripts, environment configs | Docker, GitHub Actions |\n| **supabase** (optional) | Auth (email/password, social), storage for user uploads, realtime subscriptions | Supabase |\n\n## 4. API Design (RESTful)\n- **Auth**: POST `/auth/register`, POST `/auth/login`, POST `/auth/logout`\n- **Businesses**: GET `/businesses` (search, filter, near), GET `/businesses/:id`, POST `/businesses` (provider), PATCH `/businesses/:id`, DELETE\n- **Services**: GET `/services`, GET `/services/:id`, POST `/services` (provider)\n- **Bookings**: GET `/bookings` (user), POST `/bookings`, PATCH `/bookings/:id` (cancel/reschedule), GET `/bookings/:id`\n- **Availability**: GET `/businesses/:businessId/services/:serviceId/availability?date=YYYY-MM-DD`\n- **Reviews**: GET `/reviews?businessId=:id`, POST `/reviews`\n- **Favorites**: GET `/favorites`, POST `/favorites`, DELETE `/favorites/:businessId`\n- **Notifications**: GET `/notifications`, POST `/notifications/read`\n- **Admin**: `/admin/*` (RBAC)\n- **Provider**: `/provider/*` (business owner portal)\n\nAll endpoints return JSON; validation via class-validator/pipes. Rate limiting via NestJS Throttler.\n\n## 7. State Management & Data Fetching\n- React Query handles server state: queries for businesses, services, bookings; mutations for create/update/delete.\n- Query keys are namespaced: `['businesses', filters]`, `['business', id]`, `['bookings', userId]`.\n- Background refetch intervals for availability.\n- Optimistic updates for favorites, reviews.\n- Reanimated used for gesture-based UI (e.g., swipe to delete booking, animated map markers).\n\n## 8. Caching & Background Jobs\n- Redis caches:\n  - Geospatial search results (nearby businesses) with TTL 5 min.\n  - Computed availability slots per business/service/date (TTL 10 min).\n- BullMQ workers (in API) process:\n  - Send booking confirmation emails/SMS.\n  - Send appointment reminders (scheduled via cron).\n  - Update popularity scores.\n\n## 9. Design System & Shared Types\n- `libs/ui-kit` provides primitives styled with theme tokens (colors, spacing, radius) using `react-native-reanimated` for press animations.\n- `libs/shared-types` contains TypeScript interfaces that mirror Prisma models and API DTOs, ensuring type safety across frontend and backend via code generation (optional `prisma-ts` generator).\n- Theme supports light/dark mode.\n\n## 10. DevOps & CI/CD\n- **Local**: `docker compose up` runs API, PostGIS-enabled Postgres, Redis.\n- **GitHub Actions**:\n  - Lint (ESLint, Prettier)\n  - Unit tests (Jest for both frontend and API)\n  - E2E tests (Detox for frontend, Jest for API)\n  - Build Docker images\n  - Publish to GitHub Packages (optional)\n- **EAS Build**: Configured for preview and production builds; uses `eas.json` with profiles.\n- **Supabase**: Auth via email/password and social providers; storage for user avatars and business photos; optional realtime subscriptions for booking updates.\n- **Monitoring**: API logs to stdout (captured by Docker); can be forwarded to Loki/Prometheus; frontend errors logged to Sentry (optional).\n\n## 11. Testing Strategy\n- **Unit**: Jest with React Testing Library for components; NestJS testing utilities for services/controllers.\n- **Integration**: API tests using Supertest against a test Postgres schema.\n- **E2E**: Detox for critical user flows (login, search, book appointment, view booking).\n- **Test coverage** target >80% for core domains.\n\n## 12. Scalability Considerations\n- Horizontal scaling of API via Docker Swarm/Kubernetes (stateless).\n- Read replicas for Postgres for heavy search queries.\n- Redis clustering for cache.\n- Geospatial queries leverage PostGIS GIST indexes.\n- Booking slot computation performed asynchronously; results cached.\n\n## 13.
