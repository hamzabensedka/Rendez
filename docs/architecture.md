High-Level Overview\nThe Planity Clone is a monorepo built with Nx and pnpm, comprising a mobile client (Expo/React Native) and a NestJS backend API. The mobile app uses Expo Router for navigation, TanStack React Query for data fetching, and React Native Reanimated for smooth UI interactions. The backend leverages Prisma ORM with PostgreSQL (extended by PostGIS for geospatial queries) and Redis for caching and job queuing (BullMQ). Shared TypeScript definitions, UI components, and utility libraries live in the monorepo’s libs folder. Docker Compose orchestrates local development services, while GitHub Actions, EAS Build, and Supabase (for auth & storage) handle CI/CD and deployment.\n\n## 2. Monorepo Structure\n```\nplanity-clone/\n├── apps/\n│   ├── mobile/          # Expo React Native app\n│   └── api/             # NestJS backend\n├── libs/\n│   ├── ui/              # Shared React Native components (design system)\n│   ├── types/           # Shared TypeScript interfaces & enums\n│   ├── utils/           # Cross-platform helpers (date, formatting, etc.)\n│   ├── query/           # React Query wrappers & custom hooks\n│   └── api-client/      # Generated NestJS client (via swagger or manual)\n├── prisma/\n│   ├── schema.prisma    # DB schema (PostgreSQL + PostGIS)\n│   └── migrations/\n├── docker-compose.yml\n├── nx.json\n├── package.json\n└── tsconfig.base.json\n```\n\n## 3. Mobile Application (`apps/mobile`)\n- **Entry Point**: `src/main.tsx` (Expo root).\n- **Routing**: Expo Router (`src/app/` directory‑based routing).\n- **State Management**: TanStack React Query (`src/query/`).\n- **UI Layer**: Reanimated 2 for gestures/animations (`src/animations/`).\n- **Feature Modules** (each under `src/features/`):\n  - `auth/` – login, register, password reset.\n  - `explore/` – guest browse, categories, featured businesses.\n  - `search/` – search bar, filters, map/list views.\n  - `business/` – business detail, services, pricing, reviews.\n  - `booking/` – slot selection, booking flow, confirmation.\n  - `appointments/` – upcoming, past, cancel/reschedule, reminders.\n  - `favorites/` – add/remove/view favorites.\n  - `profile/` – view/edit profile, change password.\n  - `provider/` – business owner portal (profile, services, bookings).\n  - `admin/` – admin dashboard (content, users, businesses).\n- **Services**: `src/services/` – thin wrappers around `api-client` (REST or GraphQL).\n- **Assets**: `assets/` – images, icons, fonts.\n- **Constants**: `src/constants/` – API endpoints, theme, routes.\n- **Tests**: `__tests__/` – Jest + React Native Testing Library.\n\n## 4. Backend API (`apps/api`)\n- **Framework**: NestJS (modular, CLI‑generated).\n- **Modules** (each in `src/modules/`):\n  - `auth` – JWT‑based auth (email/password, password reset).\n  - `users` – CRUD for consumer and provider profiles.\n  - `businesses` – business CRUD, geospatial search (PostGIS).\n  - `services` – service catalog, pricing, category linking.\n  - `bookings` – appointment creation, validation, slot computation.\n  - `appointments` – user‑side appointment management.\n  - `favorites` – user‑business many‑to‑many.\n  - `reviews` – review/rating CRUD, aggregation.\n  - `payments` – payment gateway integration (stubbed for Stripe/PayPal).\n  - `notifications` – email/SMS/push via Supabase functions or external provider.\n  - `provider` – business‑owner endpoints (profile, services, bookings).\n  - `admin` – admin‑only endpoints (content moderation, user/biz management).\n  - `jobs` – BullMQ processors (notification sending, slot recomputation).\n- **Prisma Service**: `src/prisma/prisma.service.ts` – singleton PrismaClient.\n- **Guards**: JWT AuthGuard, RolesGuard (USER, PROVIDER, ADMIN).\n- **Interceptors**: TransformInterceptor (response formatting), LoggingInterceptor.\n- **Validation**: DTOs with class‑validator & class‑transformer.\n- **Exceptions**: Custom HttpException filters.\n- **Testing**: Jest unit/tests, `test/` folder.\n\n## 5. Shared Libraries (`libs/`)\n- **ui**: Design system components (Button, Input, Card, Modal) built with React Native Reanimated for gestures; exported for use in mobile app.\n- **types**: Central TypeScript interfaces (User, Business, Service, Booking, Review, etc.) used by both mobile (via `api-client`) and backend (DTOs).\n- **utils**: Helper functions (date formatting, currency, geohash, password hashing).\n- **query**: Custom React Query hooks (`useBusinesses`, `useBooking`, etc.) that encapsulate query keys, stale times, and error handling.\n- **api-client**: Generated fetch client (e.g., using `openapi-typescript` or manual) that mirrors NestJS endpoints; ensures type‑safe calls from mobile.\n\n## 6. ## 9. Example Data Flows\n\n### 9.1 Guest Browse (No Auth)\n1. Mobile → `useBusinesses` query (React Query) → GET `/businesses?featured=true`\n2. API → Prisma → fetches featured businesses (cached in Redis for 5 min)\n3. Returns JSON → Mobile renders list via `ui/BusinessCard`.\n\n### 9.2 User Login\n1. Mobile → `auth/login` endpoint (POST) with email/password.\n2. API → validates credentials, compares hash, issues JWT (access + refresh).\n3. Returns tokens → Mobile stores securely (SecureStore) and sets auth header for subsequent requests.\n\n### 9.3 Booking an Appointment\n1. Mobile → `useBookingMutation` → POST `/bookings` with `{businessId, serviceId, startTime, endTime}`.\n2. API → AuthGuard validates JWT.\n3. Service layer:\n   - Checks business/service ownership.\n   - Calls Prisma to verify slot availability (via a raw SQL range‑overlap check on `bookings` table).\n   - Creates `Booking` record with status `PENDING`.\n   - Creates a BullMQ job `process-payment` (payload includes bookingId).\n4. Returns booking confirmation → Mobile shows success screen.\n5. BullMQ worker:\n   - Calls payment gateway (stub).\n   - On success: updates booking status to `CONFIRMED`, schedules reminder job.\n   - On failure: sets status `FAILED`, notifies user via email/push.\n\n### 9.4 Map‑Based Search\n1. Mobile → user moves map → debounced call to `useMapBusinesses` → GET `/businesses/search?lat=…&lon=…&radius=…&category=…`.\n2. API → Prisma query using PostGIS `ST_DWithin(location, ST_MakePoint(lon, lat)::geography, radius)`.\n3. Returns businesses within radius → Mobile renders markers via `react-native-maps`.\n\n## 10. Development Workflow\n- **Code Generation**: `nx generate @nx/react-native:component` for UI; `nest g resource` for API modules.\n- **Linting/Formatting**: `nx run-many --target=lint --all`.\n- **Testing**: `nx run-many --target=test --all`.\n- **Versioning**: Conventional commits; automated changelog via `standard-version`.\n- **Environment Variables**: Stored in `.env` (local) and GitHub Secrets (CI).\n\n## 11.
