Service Boundaries (Backend вЂ” NestJS Modules)\n\n- **AuthModule**: email/password, phone OTP (Supabase or custom), Google/Apple, JWT access + refresh, password reset. Depends on UserModule.\n- **UserModule**: profile, preferences, saved payment methods (refs Stripe). Publishes user events.\n- **BusinessModule**: CRUD business, categories, gallery, staff. Uses PostGIS for location.\n- **CategoryModule**: hierarchical taxonomy. Read-heavy.\n- **ServiceModule**: services per business, duration, price.\n- **SearchModule**: text search + filters; delegates geo to PostGIS; uses Redis for hot feeds.\n- **GeoModule**: map pins, viewport queries, `search this area`. PostGIS centered.\n- **BookingModule**: orchestrates slot select, promo, confirm, cancel, reschedule. Emits events to NotificationModule + PaymentModule.\n- **AvailabilityModule**: slot computation engine (hours, shifts, buffers, closures). Pure domain logic, cached in Redis.\n- **PaymentModule**: Stripe/PayPal intents, save card, refund. Webhooks.\n- **ReviewModule**: verified-only reviews, aggregate rating.\n- **FavoriteModule**: user favorites, synced.\n- **NotificationModule**: push (FCM/APNs) + email templates; enqueues BullMQ.\n- **JobModule**: BullMQ queues (sendNotification, expirePending, syncAvailability) on Redis.\n- **AdminModule**: approvals, global config, analytics (P2).\n\n## 4.
