  "content": "# Planity Clone — System Architecture\n\n## 1. Overview\nPlanity Clone is a mobile-first platform for beauty and wellness booking. The system is built as an Nx pnpm monorepo containing Expo (React Native) mobile and web apps, a NestJS API, and a BullMQ worker. Data is stored in PostgreSQL with PostGIS (via Supabase) and cached/queued in Redis. CI/CD uses GitHub Actions; mobile builds use EAS Build; local dev uses Docker Compose.\n\n## 2. Technology Mapping\n- **Expo + React Native + TypeScript**: Mobile client, shared web portals.\n- **Expo Router**: File-based navigation for mobile and web.\n- **TanStack React Query**: Server-state caching, pagination, mutations.\n- **React Native Reanimated**: Map sheet, transitions, micro-interactions.\n- **NestJS**: RESTful API, modular boundaries, guards, pipes.\n- **Prisma**: Type-safe DB access, migrations.\n- **PostgreSQL + PostGIS**: Relational data, geo queries.\n- **Redis**: Cache, session blacklist, BullMQ queues.\n- **Supabase**: Auth (JWT), Storage (photos), hosted Postgres, Realtime.\n- **Nx + pnpm**: Monorepo orchestration.\n- **Docker Compose**: Local Postgres-PostGIS and Redis.\n- **GitHub Actions**: Lint, test, build, deploy.\n- **EAS Build**: Cloud mobile builds.\n- **Jest**: Unit/integration tests.\n\n## 3. Folder Scaffold\n```\nplanity-clone/\n├── apps/\n│   ├── mobile/            # Expo Router iOS/Android\n│   ├── provider-portal/   # Expo web for business owners\n│   ├── admin-dashboard/   # Expo web for super admins\n│   ├── api/               # NestJS HTTP server\n│   └── worker/            # NestJS BullMQ processor\n├── libs/\n│   ├── shared/\n│   │   ├── types/         # Cross-platform TS interfaces (contracts)\n│   │   ├── ui/            # Design system (buttons, cards, sheets)\n│   │   ├── config/        # Env schemas, constants\n│   │   └── utils/         # Formatters, geo helpers\n│   ├── backend/\n│   │   ├── prisma/        # schema.prisma, migrations, seed\n│   │   ├── auth/          # Supabase JWT verification, guards\n│   │   ├── business/      # CRUD, categories, staff\n│   │   ├── search/        # PostGIS text and geo queries\n│   │   ├── availability/  # Slot computation engine\n│   │   ├── booking/       # Appointments, conflicts\n│   │   ├── payments/      # Stripe integration, refunds\n│   │   ├── reviews/       # Ratings, moderation\n│   │   ├── notifications/ # Email/SMS/Push via queues\n│   │   └── admin/         # Platform governance APIs\n│   └── mobile/\n│       ├── api/           # React Query hooks, API client\n│       ├── features/      # auth, explore, detail, booking, profile\n│       ├── components/    # Composite screens components\n│       ├── animations/    # Reanimated builders\n│       └── navigation/    # Root layout, stacks\n├── tools/                 # Nx generators, scripts\n├── docker-compose.yml     # postgis, redis\n├── nx.json\n├── package.json\n└── .github/workflows/ci.yml\n```\n\n## 4. Service Boundaries\n### API Modules (NestJS)\n- **Auth**: Validates Supabase JWT, refreshes tokens, exposes user context. No password storage (delegated to Supabase).\n- **Business**: Manages profiles, services, categories, staff, hours. Emits events on change.\n- **Search**: Accepts query, filters, sort; uses PostGIS ST_DWithin and full-text search. Returns paginated DTOs.\n- **Availability**: Pure function module computing slots from business hours, staff shifts, service duration, existing appointments. Used by Booking and Detail.\n- **Booking**: Orchestrates slot selection, creates appointment, calls Payments, enqueues Notifications.\n- **Payments**: Stripe payment intents, saved cards (tokenized), refunds. Webhook endpoint for async updates.\n- **Reviews**: Creates verified reviews post-appointment, recalculates averages via DB trigger or service method.\n- **Notifications**: Pushes jobs to Redis (BullMQ) for email/SMS/push.\n- **Favorites and Profile**: User-specific data, synced via account.\n- **Admin**: Separate guarded module for approvals, suspension, moderation.\n\n### Worker App\nConsumes BullMQ queues: `notifications`, `image-resize`, `reports`. Retry with exponential backoff; failed jobs logged.\n\n### Mobile and Web Apps\n- **mobile**: Guest browse, auth, map search, detail, booking flow, appointments, profile. Uses React Query to talk to API. Reanimated for bottom sheets and map preview.\n- **provider-portal**: Business owner CRUD, calendar, analytics. Reuses libs/shared/ui and libs/mobile/api.\n- **admin-dashboard**: Uses admin module endpoints, moderation UI.\n\n## 5. Data Layer\nPrisma schema includes:\n- `User`, `Account`, `Session` (Supabase handles auth, profile in DB)\n- `Business`, `Service`, `Category` (self-relation), `Staff`, `BusinessHours`\n- `Appointment`, `AppointmentStatus`, `Payment`\n- `Review`, `Favorite`\n- `Location` field using PostGIS (`geometry(Point, 4326)`)\n\nPostGIS enables distance filters, clustering via ST_ClusterKMeans or client-side clustering.\n\n## 6. Mobile Architecture Details\n- **Expo Router**: `app/(tabs)/home`, `app/explore`, `app/business/[id]`, `app/booking`, `app/profile`. Guest vs auth layouts.\n- **React Query**: Query keys by resource; infinite queries for search pagination; mutations for booking.\n- **Reanimated**: `BottomSheet` component, map marker preview, shared element transitions.\n- **Design System**: `libs/shared/ui` exports themed `Button`, `Card`, `Input`, `RatingStars`.\n\n## 7. Cross-Cutting Concerns\n- **Shared Types**: `libs/shared/types` defines API request/response contracts, consumed by NestJS DTOs and React Query hooks to ensure type safety.\n- **Config**: Zod schemas validate env vars in all apps.\n- **Logging and Monitoring**: NestJS interceptors; worker metrics; Sentry optional.\n- **Security**: Supabase RLS for storage; API guards for JWT; PCI via Stripe (no raw card data). Refresh token rotation in Supabase.\n\n## 8. Infrastructure\n### Local (Docker Compose)\n- `postgis/postgis` image for Postgres+PostGIS on :5432\n- `redis:7` on :6379\n- Volumes for persistence.\n\n### CI/CD (GitHub Actions)\n- On PR: `pnpm install`, `nx affected:lint`, `nx affected:test` (Jest), `nx affected:build`.\n- On merge to main: API docker build, EAS Build for mobile (preview), deploy worker.\n\n### EAS Build\n- `eas.json` with profiles `development`, `preview`, `production`. Uses Expo config from `apps/mobile`.\n\n## 9. Testing\n- **Jest** unit tests for availability engine, payments math, Prisma services with mocked client.\n- API e2e
