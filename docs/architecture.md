Reanimated.\n\n### Backend Services\n- NestJS modules represent bounded contexts:\n  * Auth Service\n  * Business Service\n  * Booking Service\n  * Notification Service\n  * Provider Service\n  * Admin Service\n  * BullMQ workers for background jobs\n\n### Data Layer\n- PostgreSQL + PostGIS for geo‑queries.\n- Prisma ORM generates TypeScript clients.\n- Redis for caching, session store, and pub/sub for real‑time updates.\n\n## Folder Structure (Nx monorepo)\n```\nplanity-clone/\n  apps/\n    mobile/                 # Expo React Native app\n    web/                    # Optional web config/                 # Shared ESLint, Prettier, TS config\n  libs/\n    auth/\n    business/\n    booking/\n    notifications/\n    provider/\n    admin/\n  infra/\n    docker-compose.yml\n services.\n- Geo‑search using PostGIS `location` point.\n- Endpoints: `/api/v1/businesses` (list, search, detail).\n- Caches frequent queries in Redis (TTL 5m).\n\n### Booking Service\n- Manages appointments, time‑slot computation, conflict detection.\n- Endpoints: `/api/v1/bookings` (create, reschedule, cancel, list).\n- Publishes availability updates to Redis channel `availability:update`.\n\n### Notification Service\n- Sends push notifications (Expo push toolkit) and emails.\n- Consumes jobs from BullMQ queue.\n\n### Provider Service\n- Allows business owners to manage listing, services, staff schedules, view bookings.\n- Built as a NestJS API consumed by a provider‑facing web app.\n\n### Admin Service\n- CRUD over users, businesses, content moderation.\n- Protected by role‑based middleware (ADMIN).\n\n### BullMQ Workers\n- Workers for: sending notifications, updating availability, cleaning expired sessions, processing payment webhooks.\n\n## Data Model Bearer <token>` header.\n- Versioning via URL prefix `/api/v1/`.\n\n## Real‑time & Caching\n- Redis stores:\n  * Session tokens (TTL 24h).\n  * Geo‑search results cache (TTL 5m).\n  * Availability slots per business (TTL 1m).\n- Pub/sub channel `availability:update` pushes new slot sets; mobile clients refetch via React Query or use a lightweight WebSocket wrapper.\n\n## Deployment & DevOps\n- **Local**: `docker compose up` starts PostgreSQL (with PostGIS), Redis, NestJS API.\n- **CI**: GitHub Actions runs lint, typecheck, unit tests (Jest), builds Docker images, pushes to registry.\n- **CD**: On main branch, images deployed to staging/prod (e.g., AWS ECS).\n- **Mobile**: EAS Build creates IPA/APK; distributed via TestFlight/Play Store internal track.\n- **Supabase**: Used for OAuth rotation.\n- Redis and PostgreSQL restricted to app network.\n\n## Scalability & Observability\n- Horizontal scale NestJS API behind load balancer.\n- Read replicas for PostGIS heavy geo queries.\n-
