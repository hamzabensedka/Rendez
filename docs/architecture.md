High-Level Overview\nPlanity Clone is a beauty & wellness booking marketplace with four client surfaces:\n- Mobile App (Expo / RN) for end clients\n- Provider Portal (web, Expo for web or Next.js-lite via Expo Router)\n- Admin Dashboard (web)\n- Background Workers (NestJS + BullMQ)\n\nAll surfaces share a single NestJS API backed by PostgreSQL + PostGIS, Redis for cache/queues, and Prisma as ORM. Monorepo managed by Nx + pnpm. Supabase used for auth/JWT and storage; EAS Build for mobile CI; GitHub Actions for pipeline.\n\n## 2. Service Boundaries\n- **Auth:** Supabase Auth issues JWT; NestJS validates via guard. Roles: client, provider, admin.\n- **Search:** `search` module uses PostGIS (`ST_DWithin`) for radius; Redis caches hot queries.\n- **Availability:** pure function module; given provider hours + bookings -> free slots. No UI.\n- **Bookings:** transactional; locks slot via DB constraint + Redis lock.\n- **Notifications:** API enqueues job; worker sends via Supabase/FCM/Twilio.\n- **Payments:** Stripe webhooks -> `payments` module -> `payout` job.\n\n## 5. Migrations via Prisma + raw SQL for GIS indexes.\n\n## 6.
