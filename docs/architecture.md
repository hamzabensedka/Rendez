Service Boundaries & Module Design\n\n### 4.1 Frontend (Mobile)\n\n- **Routing**: Expo Router with file-based routing. Auth group (`(auth)`) for unauthenticated flows, tab group (`(tabs)`) for main app.\n- **State Management**: Zustand for global state (auth, cart, filters). TanStack React Query for server state (API data caching, pagination, optimistic updates).\n- **API Layer**: Axios instance with interceptors for JWT refresh. TanStack Query hooks per feature (e.g., `useBusinesses`, `useBookSlot`).\n- **Animations**: React Native Reanimated for smooth transitions (map markers, booking flow, heart animation).\n- **Maps**: `react-native-maps` with custom clustering logic.\n\n### 4.2 Backend (NestJS)\n\nEach module follows NestJS convention: controller, service, module, DTOs, guards.\n\n| Module | Responsibility | Key Endpoints |\n|--------|----------------|---------------|\n| **Auth** | User registration, login, OAuth, phone verification, JWT management | `POST /auth/signup`, `POST /auth/login`, `POST /auth/refresh` |\n| **Users** | Profile management, roles, favorites | `GET /users/me`, `PATCH /users/me`, `POST /users/favorites` |\n| **Businesses** | CRUD for business profiles, services, team members, images | `GET /businesses`, `GET /businesses/:id`, `POST /businesses` |\n| **Categories** | Category tree, featured categories | `GET /categories`, `GET /categories/:id/subcategories` |\n| **Search** | Full-text search with Elasticsearch, geospatial with PostGIS, auto-complete | `GET /search?q=&lat=&lng=&radius=` |\n| **Bookings** | Slot availability, booking creation, cancellation, calendar sync | `GET /bookings/slots`, `POST /bookings`, `DELETE /bookings/:id` |\n| **Reviews** | Create, read, update reviews; rating aggregation | `GET /businesses/:id/reviews`, `POST /reviews` |\n| **Payments** | Stripe integration, payment intents, refunds | `POST /payments/intent`, `POST /payments/confirm` |\n| **Notifications** | Push (Expo Push API), email (SendGrid), SMS (Twilio) | `POST /notifications/send` |\n| **Admin** | User management, business approval, analytics | `GET /admin/dashboard`, `PATCH /admin/businesses/:id/approve` |\n\n### 4.3 Data Layer\n\n- **PostgreSQL** with PostGIS: Primary database for all transactional data. PostGIS enables distance queries and map bounds filtering.\n- **Redis**: Session cache, rate limiting, temporary data (SMS codes, password reset tokens), and caching for frequent queries (search results, business details).\n- **Elasticsearch**: Full-text search with fuzzy matching, auto-complete, and trending searches. Indexed from PostgreSQL via CDC or scheduled sync.\n\n## 5. Data Flow & Key Interactions\n\n### 5.1 Booking Flow\n\n1. Client searches for services (Elasticsearch + PostGIS).\n2. Client views business detail (cached in Redis, fallback to DB).\n3. Client selects service and professional.\n4. Client requests available slots (Redis cache of precomputed slots, DB for real-time).\n5. Client selects time, adds-ons, and proceeds to payment.\n6. Payment intent created via Stripe (idempotency key).\n7. On success, booking record created in PostgreSQL, slot marked unavailable.\n8. Notifications sent to client and provider (push, email).\n9. Calendar event generated (optional).\n\n### 5.2 Authentication Flow\n\n1. User signs up (email/password or OAuth).\n2. Backend validates, creates user in DB, returns JWT access + refresh tokens.\n3. Mobile stores tokens securely (expo-secure-store).\n4. Axios interceptor attaches access token to requests.\n5. On 401, interceptor calls `/auth/refresh` with refresh token.\n6. If refresh fails, user is logged out.\n\n## 6. Testing Strategy\n\n- **Unit Tests**: Jest for both mobile and API. Security & Performance\n\n- **Security**: JWT with short expiry, refresh token rotation, rate limiting (Redis), input validation (class-validator), Helmet, CORS.\n- **Performance**: Redis caching for frequent reads, database indexing (GiST for PostGIS, B-tree for foreign keys), pagination, lazy loading images.\n\n## 9.
