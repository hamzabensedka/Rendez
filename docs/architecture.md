High-Level Overview\nPlanity Clone is a mobile-first beauty & wellness booking marketplace. The system is split into:\n- **Mobile App** (Expo + RN): customer-facing iOS/Android.\n- **Provider Portal** (Expo Web): business owner management.\n- **Admin Dashboard** (Expo Web): internal ops.\n- **API Backend** (NestJS): core business logic, auth, booking, search.\n- **Workers** (NestJS + BullMQ): notifications, reminders, cache jobs.\n- **Data Layer**: PostgreSQL + PostGIS, Redis (cache/queue), Supabase (Auth/Storage baseline).\n\nMonorepo managed by **Nx** with **pnpm**. CI/CD via GitHub Actions, build via EAS Build, local via Docker Compose.\n\n## 2. Service Boundaries\n### 3.1 Mobile App (apps/mobile)\n- Expo Router tabs/stacks: `(guest)`, `(auth)`, `(app)`.\n- Screens use TanStack React Query for server state; Reanimated for transitions.\n- No business logic; calls API via `api-client`.\n\n### 3.2 Provider Portal (apps/provider-portal)\n- Expo Web with same UI lib.\n- CRUD businesses, services, staff, hours, view bookings.\n\n### 3.3 Admin (apps/admin)\n- Expo Web; user/provider/category management, metrics.\n\n### 3.4 API (apps/api)\nNestJS modules:\n- `AuthModule`: JWT, roles, Supabase email verify.\n- `UserModule`: profiles, addresses.\n- `BusinessModule`: CRUD, categories.\n- `SearchModule`: text + PostGIS map search.\n- `BookingModule`: slots, availability, appointments.\n- `ReviewModule`: ratings (P1).\n- `PaymentModule`: Stripe (P1).\n- `NotificationModule`: email/SMS triggers.\n- `AdminModule`: admin ops.\n\n### 3.5 Worker (apps/worker)\n- BullMQ queues in Redis: `email`, `sms`, `reminder`, `slot-cache`.\n- Consumes domain events from API.\n\n### 3.6 Data\n- PostgreSQL + PostGIS: geo queries.\n- Redis: query cache, session blacklist, BullMQ.\n- Prisma: schema, migrations.\n\n## 4.
