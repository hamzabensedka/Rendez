The system is built as an Nx monorepo managed by pnpm, containing a NestJS API, Expo React Native mobile app, web portals, and shared libraries. Supabase provides hosted PostgreSQL (with PostGIS), Auth, and Storage. Redis backs BullMQ background jobs and caching.\n\n## 2. Backend Architecture (NestJS)\nThe API is a stateless NestJS service. It uses Prisma as ORM over Supabase PostgreSQL (with PostGIS extension). Redis for BullMQ and caching.\n\n### Service Boundaries (Modules)\n- **AuthModule**: email/phone OTP/social login, JWT access + refresh, Supabase Auth integration, token revocation.\n- **UserModule**: profile, payment methods (tokenized), notification settings.\n- **BusinessModule**: CRUD business profiles, staff, categories, photos (Supabase Storage).\n- **ServiceModule**: service catalog per business, durations, prices.\n- **CategoryModule**: taxonomy seed/management (admin).\n- **AvailabilityModule**: uses libs/availability-engine to compute slots; respects timezone, lead time, breaks.\n- **BookingModule**: cart, slot locking (Redis distributed lock), create/cancel/reschedule, policy enforcement.\n- **PaymentModule**: integrates Stripe/Adyen, 3DS, refunds, invoice email trigger.\n- **NotificationModule**: enqueues BullMQ jobs for push (Firebase/APNs), email, SMS; respects quiet hours.\n- **ReviewModule**: verified-only reviews, avg rating update, provider responses.\n- **FavoriteModule**: user bookmarks.\n- **SearchModule**: text + geo search via PostGIS; filter combining; history.\n- **AdminModule**: business approval, user disable, queue monitoring, audit logs.\n- **WorkerModule**: BullMQ processors for reminders, email, cache warming, analytics.\n\n### Data Layer\n- **Prisma schema** defines models: User, Business, Staff, Service, Category, Appointment, Review, Favorite, Payment, RefreshToken, SearchHistory.\n- **PostGIS**: Business has location geography(Point,4326). Spatial indexes for map search.\n- **Redis**: \n  - booking_lock:{businessId}:{slot} for concurrency.\n  - search_cache:{queryHash} for <500ms p95.\n  - BullMQ queues: notifications, reminders, emails, analytics.\n\n### Availability Engine\nPure TS lib libs/availability-engine:\n- Input: business hours, staff schedules, service duration, existing bookings, timezone, lead time.\n- Output: 15-min granularity slots, excluding breaks/days off.\n- Used by API and tested with Jest.\n\n## 4. Mobile App Architecture (Expo)\n- **Expo Router** file-based routing under apps/mobile/app/.\n- **TanStack React Query** for server state, caching, mutations; custom queryClient with retry/backoff.\n- **React Native Reanimated** for smooth transitions (bottom sheets, map cards, tab animations).\n- **Supabase Auth** session bridged to React Query; secure storage for tokens (Expo SecureStore).\n- Feature folders: features/auth, features/explore, features/search, features/booking, features/profile, features/map.\n- API communication via @planity/shared typed clients.\n\n### Mobile Folder Structure\n```\napps/mobile/\nв”њв”Ђв”Ђ app/\nв”‚   в”њв”Ђв”Ђ (tabs)/\nв”‚   в”‚   в”њв”Ђв”Ђ home.tsx\nв”‚   в”‚   в”њв”Ђв”Ђ search.tsx\nв”‚   в”‚   в”њв”Ђв”Ђ bookings.tsx\nв”‚   в”‚   в””в”Ђв”Ђ profile.tsx\nв”‚   в”њв”Ђв”Ђ business/[id].tsx\nв”‚   в”њв”Ђв”Ђ booking/\nв”‚   в”‚   в”њв”Ђв”Ђ service.tsx\nв”‚   в”‚   в”њв”Ђв”Ђ slot.tsx\nв”‚   в”‚   в””в”Ђв”Ђ confirm.tsx\nв”‚   в”њв”Ђв”Ђ login.tsx\nв”‚   в””в”Ђв”Ђ _layout.tsx\nв”њв”Ђв”Ђ components/\nв”њв”Ђв”Ђ features/\nв”‚   в”њв”Ђв”Ђ auth/\nв”‚   в”њв”Ђв”Ђ explore/\nв”‚   в”њв”Ђв”Ђ search/\nв”‚   в”њв”Ђв”Ђ map/\nв”‚   в”њв”Ђв”Ђ booking/\nв”‚   в””в”Ђв”Ђ profile/\nв”њв”Ђв”Ђ hooks/\nв”њв”Ђв”Ђ api/                # React Query hooks\nв”њв”Ђв”Ђ theme/\nв”њв”Ђв”Ђ assets/\nв””в”Ђв”Ђ jest.config.js\n```\n\n## 5.
