  "content": "# Planity Clone System Architecture\n\n## 1. Overview\nPlanity Clone is a mobile-first service discovery and booking platform. The architecture follows a clean, layered approach with clear separation of concerns: presentation (Expo/React Native), application/services (NestJS), data (Prisma + PostgreSQL/PostGIS + Redis), and shared tooling (Nx monorepo, pnpm, Docker, GitHub Actions, EAS Build).\n\n## 2. Technology Stack\n- **Mobile**: Expo (SDK 49), React Native, TypeScript, Expo Router, TanStack React Query, React Native Reanimated\n- **Backend**: NestJS (Node.js), TypeScript, Prisma ORM, PostgreSQL, PostGIS extension, Redis, BullMQ (for background jobs)\n- **DevOps**: Docker Compose (local dev), GitHub Actions (CI/CD), EAS Build (Expo builds), Supabase (optional hosted PG/Redis for staging)\n- **Monorepo**: Nx workspace, pnpm\n- **Testing**: Jest (unit/integration), React Native Testing Library, Supertest\n\n## 3. HighвЂ‘Level Architecture\n```\n+-------------------+        +---------------------+        +-------------------+\n|   Mobile App      | <----> |   API Gateway (Nest) | <----> |   Data Store      |\n| (Expo/RN)         |        |   (REST/GraphQL)    |        | (Postgres/PostGIS)|\n+-------------------+        +---------------------+        +-------------------+\n        ^                           ^                         ^\n        |                           |                         |\n        |                           |                         |\n        |                           |                         |\n        |                           +-------------------+        +-------------------+\n                           |   Shared Libs     |        |   Infra (Redis)   |\n                           | (types, utils)    |        |   (caching, jobs) |\n                           +-------------------+        +-------------------+\n```\n- **Mobile App** consumes the NestJS REST API (optionally GraphQL via Apollo). State managed with React Query; UI animations with Reanimated.\n- **API Gateway** (NestJS) handles authentication, validation, business logic, and delegates to services.\n- **Data Store**: Primary relational DB (PostgreSQL + PostGIS) for entities; Redis for caching, session store, and BullMQ job queue.\n- **Shared Libraries** (Nx libs) contain TypeScript interfaces, DTOs, utility functions, and design system tokens used by both mobile and backend.\n\n## 4. Service Boundaries (Backend)\n| Service          | Responsibility                                   | Key Endpoints (examples)                              |\n|------------------|--------------------------------------------------|-------------------------------------------------------|\n| **Auth**         | User registration, login, JWT issuance, refresh  | `POST /auth/register`, `POST /auth/login`, `POST /auth/refresh` |\n| **Users**        | Profile CRUD, password change, preferences       | `GET /users/me`, `PATCH /users/me`, `POST /users/me/password` |\n| **Businesses**   | Business CRUD, search, filtering, geoвЂ‘queries    | `GET /businesses`, `GET /businesses/:id`, `POST /businesses` |\n| **Services**     | Service catalog tied to businesses               | `GET /services`, `GET /services/:id`, `POST /businesses/:id/services` |\n| **Bookings**     | Slot computation, booking creation, lifecycle    | `GET /bookings`, `POST /bookings`, `PATCH /bookings/:id/cancel`, `PATCH /bookings/:id/reschedule` |\n| **Favorites**    | Favorite businesses management                   | `GET /favorites`, `POST /favorites`, `DELETE /favorites/:id` |\n| **Reviews**      | Ratings & reviews CRUD, aggregation              | `GET /reviews/:businessId`, `POST /reviews`, `PATCH /reviews/:id` |\n| **Payments**     | Payment gateway integration (Stripe/PayPal)      | `POST /payments/create-intent`, `POST /payments/webhook` |\n| **Notifications**| Template storage, dispatch via BullMQ workers    | Internal (triggered by other services)                |\n| **Admin**        | AdminвЂ‘only CRUD, settings, analytics             | `GET /admin/stats`, `PATCH /admin/settings`           |\n| **Provider Portal**| Business owner scoped CRUD, booking view       | `GET /provider/bookings`, `PATCH /provider/business`  |\n\nEach service is a NestJS module with its own controller, service, DTOs, and entities. CrossвЂ‘cutting concerns (validation, logging, error handling, auth guards) are implemented as NestJS interceptors/guards/global pipes.\n\n## 5. Data Modeling (Prisma)\n- **User** (id, email, passwordHash, role, createdAt, updatedAt)\n- **Business** (id, name, description, address, lat/lng (PostGIS Point), phone, website, createdAt, updatedAt)\n- **BusinessHours** (id, businessId, dayOfWeek, openTime, closeTime, isClosed)\n- **Service** (id, businessId, name, description, duration, price, createdAt, updatedAt)\n- **Booking** (id, userId, serviceId, startTime, endTime, status, paymentId, createdAt, updatedAt)\n- **Favorite** (id, userId, businessId, createdAt)\n- **Review** (id, businessId, userId, rating, comment, createdAt, updatedAt)\n- **Payment** (id, bookingId, gateway, amount, currency, status, transactionId, createdAt)\n- **Notification** (id, userId, type, payload, read, sentAt, createdAt)\n\nIndexes: spatial index on Business.location, composite indexes on (userId, status) for Bookings, (businessId) for Reviews.\n\n## 6. Caching & Background Jobs\n- **Redis** stores:\n  - Session tokens (optional JWT blacklist)\n  - Cached business listings & search results (TTL 5вЂ‘15 min)\n  - Computed availability slots per business/date (TTL 10 min)\n- **BullMQ** queues:\n  - `email-notifications` (send confirmation, reminders)\n  - `sms-notifications` (optional)\n  - `slot-recalc` (triggered on booking/create/cancel/update)\n  Workers are NestJS services processed via `@nestjs/bullmq`.\n\n## 7. Mobile App Architecture\n### 7.1 Layered Structure\n```

- **Routing**: Expo Router provides fileвЂ‘based navigation; auth gate via a wrapper layout that redirects unauthenticated users to login/signup.\n- **State**: React Query handles server cache; mutations for write operations (login, booking, etc.).\n- **Animations**: Reanimated 2 for gestureвЂ‘based UI (e.g., swipeable cards, map markers).\n- **Design System**: Tokens (spacing, color, typography) stored in an Nx lib (`ui-design-system`) and consumed via a ThemeProvider.\n- **Geo**: `expo-location` + `react-native-maps`; map queries sent to backend with bbox or radius.\n\n## 8. DevOps & CI/CD\n### 9.1 Local Development\n- Docker Compose spins up postgres (with PostGIS), redis, and the NestJS API.\n- Expo development client runs via `expo start`.\n\n### 9.2 CI (GitHub Actions)\n```yaml

Testing Strategy\n- **Unit**: Jest + React Native Testing Library for components; NestJS testing utilities for services/controllers.\n- **Integration**: Supertest against API endpoints with test DB (PostgreSQL in Docker).\n- **E2E** (optional): Detox for critical flows (login в†’ search в†’ book).\n- **Coverage**: Enforce >80% via nx affected:test.\n\n## 11. Security Considerations\n- Authentication: JWT (access token 15вЂЇmin, refresh token 7вЂЇdays) stored in SecureStore.\n- Passwords: bcrypt hash (salt 12).\n- API: Rate limiting (express-rate-limit), helmet, CORS restricted to mobile app domains.\n- Data: Parameterized queries via Prisma; SQL injection mitigated.\n- Payments: PCIвЂ‘SAQвЂ‘D compliance via Stripe; webhook signature verification.\n- Secrets: Managed via GitHub Secrets and injected into Docker/EAS env.\n\n## 12.
