  "content": "# System Architecture for Planity Clone\n\n## Overview\nThe Planity Clone is a mobile‑first application that connects users with local service businesses for discovery, booking, and management. The architecture follows a clean, modular approach with clear separation of concerns between the mobile client, backend services, shared libraries, and infrastructure. A monorepo managed by Nx and pnpm enables code sharing and consistent tooling.\n\n## Technology Stack\n- **Mobile Client**: Expo, React Native, TypeScript, Expo Router (file‑based navigation), TanStack React Query (server state), React Native Reanimated (gestures & animations)\n- **Backend API**: NestJS (modular, TypeScript), Prisma ORM, PostgreSQL with PostGIS extension for geo‑queries, Redis (caching & pub/sub), BullMQ (background job processing)\n- **Shared Libraries**: UI component kit (design system), shared TypeScript interfaces, utility helpers, ESLint/Prettier configs\n- **DevOps & CI/CD**: Nx monorepo, pnpm workspace, Docker Compose for local development, GitHub Actions for lint/test/build, EAS Build for mobile binaries, Supabase for authentication (optional) and object storage, Nginx reverse proxy (optional)\n\n## High-Level Architecture\n```\n[Mobile App] <--HTTPS/WS--> [API Gateway (NestJS)] <---> [PostgreSQL + PostGIS]\n                                   |                |\n                                   |                +--> [Redis] (cache, pub/sub)\n                                   |\n                                   +--> [BullMQ Workers] (jobs: notifications, slot computation, payment webhook handling)\n```\nThe mobile app communicates exclusively with the NestJS backend via RESTful endpoints (could be extended to GraphQL). The backend handles business logic, data persistence, geo‑search via PostGIS, caching, and dispatches background jobs through BullMQ. Workers run in separate Node processes (or same process in dev) and interact with Redis for job queues.\n\n## Folder Structure\n```\nplanity-clone/\n├── apps/\n│   ├── mobile/          # Expo React Native client\n│   │   ├── src/\n│   │   │   ├── assets/\n│   │   │   ├── components/\n│   │   │   ├── navigation/   #
