High‑Level Architecture\n```\n+---------------------+        +---------------------+        +---------------------+\n|   Mobile App (Expo) | <----> |   API Gateway (Nest) | <----> |   Data Store (PG)   |\n+---------------------+        +---------------------+        +---------------------+\n        ^                              ^                         ^\n        |                              |                         |\n        |               +--------------+--------------+         |\n        |               |                           |         |\n        v               v                           v         v\n+---------------------+   +---------------------+   +---------------------+\n|   Auth Service      |   |   Business Service  |   |   Booking Service   |\n+---------------------+   +---------------------+   +---------------------+\n        |                           |                           |\n        v                           v                           v\n+---------------------+   +---------------------+   +---------------------+\n|   Redis Cache       |   |   PostGIS (Geo)     |   |   BullMQ Workers    |\n+---------------------+   +---------------------+   +---------------------+\n```\nThe mobile app communicates exclusively with a NestJS backend via REST/GraphQL (REST chosen for simplicity). The backend splits into bounded contexts (services) that share a Prisma‑generated client. Redis provides caching, session store, and pub/sub for real‑time updates. BullMQ processes asynchronous jobs (notifications, slot recomputation). PostGIS enables location‑based queries.\n\n## 4. Data Models (Prisma Schema)\n```prisma\n// Prisma schema (libs/shared-types/src/generated/schema.prisma)\nmodel User {\n  id            String   @id @default(uuid())\n  email         String   @unique\n  passwordHash  String\n  name          String\n  role          Role     @default(USER)\n  createdAt     DateTime @default(now())\n  updatedAt     DateTime @updatedAt\n  bookings      Booking[]\n  favorites     Favorite[]\n  reviews       Review[]\n}\nmodel Business {\n  id            String   @id @default(uuid())\n  name          String\n  description   String?\n  address       String\n  location      Point    @db.Point\n  categoryId    String\n  category      Category @relation(fields: [categoryId], references: [id])\n  hours         BusinessHours[]\n  services      Service[]\n  bookings      Booking[]\n  reviews       Review[]\n  favorites     Favorite[]\n  createdAt     DateTime @default(now())\n  updatedAt     DateTime @updatedAt\n}\nmodel Category { id String @id @default(uuid()) name String @unique businesses Business[] }\nmodel Service { id String @id @default(uuid()) name String price Int durationInt Int businessId String business Business @relation(fields: [businessId], references: [id]) bookings Booking[] }\nmodel Booking { id String @id @default(uuid()) userId String user User @relation(fields: [userId], references: [id]) serviceId String service Service @relation(fields: [serviceId], references: [id]) startDateTime DateTime endDateTime DateTime status BookingStatus @default(PENDING) amount Int paymentId String? Design System & Shared Types\n- `libs/design-system` provides primitives: Button, Input, Card, Avatar, Badge, Loader.\n- Theming uses `react-native-paper`‑like dark/light tokens; colors exported from `theme/tokens.ts`.\n- Animations use Reanimated 2 for shared‑element transitions (e.g., list → detail).\n- Shared types (`libs/shared-types`) are generated via `prisma generate` and consumed to ensure compile‑time safety between client and server.\n\n## 11.
