Overview\nPlanity Clone is a mobile-first beauty & wellness booking marketplace. It consists of:\n- **Mobile App** (Expo + React Native) for consumers and providers (portal subset).\n- **Backend API** (NestJS) with Prisma + PostgreSQL/PostGIS and Redis (cache + BullMQ).\n- **Shared package** for types and design system.\n- **Infra** via Docker Compose, CI/CD via GitHub Actions, EAS Build for mobile, Supabase for auth/email optional.\n\nMonorepo managed by **Nx** with **pnpm** workspaces.\n\n## 2. Monorepo Layout (Nx + pnpm)\n```\nplanity-clone/\n├── apps/\n│   ├── mobile/                # Expo React Native app (consumer + provider)\n│   ├── api/                   # NestJS backend\n│   ├── admin-web/             # (P1) Admin dashboard (Next or Expo web)\n│   └── provider-web/          # (P0) Provider portal (Expo web or Next)\n├── libs/\n│   ├── shared-types/          # TS interfaces (API + UI)\n│   ├── ui/                    # Design system (RN components)\n│   ├── config/                # ESLint, TS, jest presets\n│   └── utils/                 # Format, geo helpers\n├── services/\n│   └── worker/                # BullMQ processor (within api or standalone)\n├── infra/\n│   ├── docker/                # Dockerfiles, docker-compose.yml\n│   ├── ci/                    # GitHub Actions workflows\n│   └── supabase/              # migrations, config\n├── docs/\n│   ├── product.md\n│   └── architecture.md\n├── package.json\n├── pnpm-workspace.yaml\n├── nx.json\n└── tsconfig.base.json\n```\n\n## 3. Mobile App (apps/mobile)\n```\napps/mobile/\n├── app/                       # Expo Router routes\n│   ├── (public)/\n│   │   ├── index.tsx          # Guest home\n│   │   ├── search.tsx\n│   │   ├── map.tsx\n│   │   └── business/[id].tsx\n│   ├── (auth)/\n│   │   ├── login.tsx\n│   │   ├── register.tsx\n│   │   └── reset.tsx\n│   ├── (app)/\n│   │   ├── favorites.tsx\n│   │   ├── profile.tsx\n│   │   ├── appointments/\n│   │   │   ├── index.tsx\n│   │   │   └── [id].tsx\n│   │   └── booking/\n│   │       └── [businessId].tsx\n│   ├── provider/              # Provider portal screens\n│   │   ├── dashboard.tsx\n│   │   ├── services.tsx\n│   │   └── schedule.tsx\n│   └── _layout.tsx\n├── src/\n│   ├── api/                   # React Query hooks\n│   ├── components/            # Feature components\n│   ├── animations/            # Reanimated utils\n│   ├── store/                 # Auth/session state\n│   └── lib/                   # Supabase client, tokens\n├── assets/\n├── eas.json\n└── package.json\n```\n\n## 4. Backend API (apps/api)\n```\napps/api/\n├── src/\n│   ├── main.ts\n│   ├── app.module.ts\n│   ├── modules/\n│   │   ├── auth/              # JWT, signup, OAuth\n│   │   ├── users/\n│   │   ├── businesses/        # CRUD, categories\n│   │   ├── categories/\n│   │   ├── services/          # Service offerings\n│   │   ├── staff/\n│   │   ├── availability/      # Slot computation\n│   │   ├── bookings/          # Create/cancel/reschedule\n│   │   ├── reviews/           # P1\n│   │   ├── payments/          # P1 Stripe\n│   │   ├── notifications/     # Email/push\n│   │   ├── favorites/         # P1\n│   │   └── admin/             # P1 guard + ops\n│   ├── common/               # Guards, decorators, pipes\n│   ├── jobs/                 # BullMQ queues\n│   ├── prisma/               # PrismaService, migrations\n│   └── redis/                # RedisModule\n├── prisma/\n│   ├── schema.prisma         # PG + PostGIS extensions\n│   └── seed.ts\n├── test/                     # Jest e2e\n└── Dockerfile\n```\n\n## 5. Shared Libraries\n- **libs/shared-types**: `User`, `Business`, `Service`, `Booking`, `Slot`, `Review`, API DTOs. Imported by mobile and api.\n- **libs/ui**: Button, Card, MapPin, Sheet (Reanimated), Theme tokens. Consumed by mobile/provider/admin.\n- **libs/config**: Base jest, eslint, tsconfig.\n\n## 6. Service Boundaries\n| Domain | Owner | Responsibilities | Depends On |\n|--------|-------|------------------|------------|\n| Auth | api/auth | Register, login, JWT, reset | users, redis |\n| Discovery | api/businesses, categories | List, search, map query (PostGIS) | redis cache |\n| Booking | api/bookings, availability | Slot gen, conflict check, persist | businesses, staff, redis |\n| Provider | api/businesses, services, staff | Portal data ops | auth (role) |\n| Notifications | api/notifications + jobs | Queue email/push | redis, bookings events |\n| Payments | api/payments | Stripe charge, rollback | bookings |\n| Admin | api/admin | Moderate, metrics | all |\n\nMobile never calls DB; only API via React Query. Provider/Admin are separate apps but share API.\n\n## 7. Data Model (Prisma + PostGIS)\n- `User(id, email, phone, role, geom?)`\n- `Business(id, name, categoryId, geom geography(Point), address, ownerId)`\n- `Category(id, name, parentId?)`\n- `Service(id, businessId, duration, price)`\n- `Staff(id, businessId)`\n- `AvailabilityRule(id, staffId/businessId, day, start, end)`\n- `Booking(id, userId, businessId, serviceId, staffId?, start, end, status)`\n- `Review(id, bookingId, stars, text)`\n- `Favorite(userId, businessId)`\n\nPostGIS used for `geom` and distance queries (`ST_DWithin`).\n\n## 8. Key Flows\n1. **Search/Map**: API `/businesses?near=lat,lng&cat=` → PostGIS query → Redis cache 5m → RN MapView pins.\n2. **Booking**: `/availability?business=&date=` computes slots (exclude booked + breaks) → confirm POST `/bookings` in transaction.\n3. **Notifications**: Booking created → BullMQ job → Supabase email / Expo push.\n\n## 9. Infra & CI\n- **Docker Compose**: postgres (postgis), redis, api.\n- **GitHub Actions**: pnpm install, Nx affected test/build, api docker push, mobile EAS build on tag.\n- **Supabase**: Auth email templates, storage for photos (optional).\n- **EAS Build**: profiles for dev/preview/prod.\n\n## 10. Testing\n- Jest unit in libs/api.\n- E2E: NestJS supertest + Prisma test DB.\n- Mobile: Jest + React Native Testing Library.\n\n## 11. Priorities Mapping\nP0: auth, guest, search, map, detail, categories, booking, appointments, availability, shared, notifications(basic), provider portal.\nP1: favorites, reviews, payment, profile, admin, jobs.\nP2: advanced notifs, analytics.\n\nThis architecture enforces separation of concerns, scalable backend, and reusable cross-platform UI.",
