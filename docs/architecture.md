It supports customers (mobile), providers (web dashboard), admins (web dashboard), and background jobs. The system is built as an Nx monorepo managed by pnpm, with clear separation between client apps, shared libraries, and a NestJS backend.\n\n## 2. Goals & Principles\n- Clean architecture: separation of concerns between UI, domain, and infrastructure.\n- Type safety end-to-end via TypeScript and shared libs/types.\n- Offline-friendly mobile with TanStack React Query caching.\n- Geo-aware search using PostGIS.\n- Async workloads via Redis/BullMQ.\n- Maintainable monorepo with Nx project boundaries.\n\n## 3. Technology Stack\n- Mobile & Web Clients: Expo, React Native, Expo Router, TypeScript, TanStack React Query, React Native Reanimated.\n- Backend: NestJS, Prisma, PostgreSQL + PostGIS, Redis (BullMQ, cache).\n- Infra: Supabase (managed Postgres/PostGIS, Storage), Docker Compose, GitHub Actions, EAS Build.\n- Tooling: Nx, pnpm, Jest.\n\n## 4.
