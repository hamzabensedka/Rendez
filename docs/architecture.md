  "content": "# Planity Clone — System Architecture\n\n## 1. Overview\nPlanity Clone is a mobile-first platform for booking beauty & wellness services. This document defines a clean, scalable architecture using a pnpm + Nx monorepo, Expo/React Native mobile client, NestJS backend, Prisma + PostgreSQL/PostGIS, Redis/BullMQ, Supabase (managed Postgres & Auth), and CI/CD via GitHub Actions, Docker Compose, and EAS Build.\n\n## 2. Tech Stack\n- **Mobile**: Expo, React Native, TypeScript, Expo Router, TanStack React Query, React Native Reanimated.\n- **Backend**: NestJS, Prisma, PostgreSQL, PostGIS, Redis (BullMQ), Supabase (Auth + Storage).\n- **Tooling**: Nx, pnpm, Docker Compose, GitHub Actions, EAS Build, Jest.\n\n## 3. Monorepo Layout (Nx + pnpm)\nWe use a single monorepo with clear app/lib boundaries.\n\n```\nplanity-clone/\n├── apps/\n│   ├── mobile/                # Expo React Native app (iOS/Android/Web)\n│   ├── api/                   # NestJS HTTP + Queue worker\n│   ├── provider-portal/      # React (Vite) or Expo web for business owners\n│   └── admin-dashboard/       # React (Vite) or Expo web for super admins\n├── libs/\n│   ├── shared-types/          # TS interfaces, DTOs, enums (used by all apps)\n│   ├── ui/                    # RN component library (Button, Card, Input)\n│   ├── design-system/         # Theme, colors, typography, spacing\n│   ├── api-client/            # React Query hooks, fetch wrappers\n│   ├── geospatial/            # PostGIS query builders, distance utils\n│   ├── queue/                 # BullMQ producer/consumer shared config\n│   ├── prisma/                # Prisma schema, migrations, client\n│   └── backend/\n│       ├── auth/              # Auth module (JWT, Supabase sync)\n│       ├── users/             # User profile module\n│       ├── businesses/        # Business & staff module\n│       ├── categories/        # Service categories module\n│       ├── services/          # Service catalog module\n│       ├── search/            # Geo-search & filters (PostGIS)\n│       ├── availability/      # Slot computation engine\n│       ├── bookings/          # Appointment lifecycle\n│       ├── payments/          # Stripe/PayPal integration\n│       ├── reviews/           # Ratings & reviews\n│       ├── notifications/     # Email/push dispatch\n│       ├── favorites/         # User favorites\n│       └── admin/             # Admin controls\n├── tools/\n│   ├── scripts/               # Seed, migrations, codegen\n│   └── eslint-rules/          # Custom lint\n├── docker-compose.yml         # Postgres+PostGIS, Redis, API\n├── nx.json\n├── package.json\n├── pnpm-workspace.yaml\n├── tsconfig.base.json\n└── .github/\n    └── workflows/\n        ├── ci.yml             # Lint, test, build (Nx affected)\n        ├── mobile.yml         # EAS Build on release\n        └── deploy.yml         # Docker compose / Supabase\n```\n\n## 4. Service Boundaries\n\n### 4.1 Mobile App (`apps/mobile`)\nFeature-first structure under `src/app` (Expo Router):\n- `auth/` – login, register, social, reset.\n- `home/` – guest browse, featured, categories.\n- `search/` – text search, filters.\n- `map/` – Reanimated map interactions, pins.\n- `business/[id]/` – detail, services, reviews, booking CTA.\n- `booking/` – multi-step flow (service, staff, slot, pay).\
