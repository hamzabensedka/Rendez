Service Boundaries & Modules\n\n### 4.1 Backend Modules (NestJS)\n\n| Module | Responsibility | Key Entities |\n|--------|----------------|--------------|\n| **AuthModule** | User registration, login, OAuth, password reset, token management | User, RefreshToken, OAuthProvider |\n| **BusinessModule** | Business profiles, categories, services, team management | Business, Service, Professional, Category |\n| **SearchModule** | Full-text search, geospatial queries, autocomplete, filtering | (uses Business, Service) |\n| **BookingModule** | Appointment lifecycle, slot management, waitlist, calendar sync | Appointment, Slot, WaitlistEntry |\n| **ReviewModule** | Ratings, reviews, moderation | Review, ReviewReport |\n| **PaymentModule** | Payment processing, refunds, payout to providers | Transaction, Payout, PaymentMethod |\n| **NotificationModule** | Push, SMS, email notifications, templates | Notification, NotificationTemplate |\n| **AdminModule** | Platform management, analytics, user moderation | Admin, AuditLog |\n\n### 4.2 Mobile Feature Modules\n\n| Feature | Key Screens | Dependencies |\n|---------|-------------|--------------|\n| **Auth** | Login, Register, ForgotPassword, OTP | TanStack Query, Supabase Auth |\n| **Search** | SearchBar, FilterSheet, ResultsList, MapView | Reanimated, MapLibre |\n| **Business** | BusinessDetail, ServiceList, ProfessionalList | TanStack Query, Reanimated |\n| **Booking** | ServiceSelection, DateTimePicker, Confirmation | TanStack Query, Reanimated |\n| **Appointments** | UpcomingList, PastList, AppointmentDetail | TanStack Query, Calendar |\n| **Favorites** | FavoritesList | TanStack Query |\n| **Profile** | UserProfile, Settings, PaymentMethods | TanStack Query |\n\n## 5. Data Flow & Communication\n\n### 5.1 API Communication\n- **Protocol**: REST over HTTPS (primary), WebSocket (real-time availability)\n- **Client**: Axios instance with interceptors for token refresh, error handling\n- **Caching**: TanStack React Query with stale-while-revalidate strategy\n- **Real-time**: Supabase Realtime for slot availability, booking confirmations\n\n### 5.2 Authentication Flow\n1. Client sends credentials to `/auth/login`\n2. Server validates, returns access token (15 min) and refresh token (7 days)\n3. Access token stored in memory, refresh token in SecureStore (Expo)\n4. Axios interceptor checks token expiry, refreshes automatically\n5. Biometric login uses stored refresh token to obtain new access token\n\n### 5.3 Booking Flow\n1. Client fetches available slots via `GET /businesses/:id/slots?date=...`\n2. User selects slot, client sends `POST /bookings` with optimistic UI\n3. Server creates booking with 5-minute hold (Redis TTL)\n4. Payment processed if required, then booking confirmed\n5. Real-time update sent to provider via WebSocket\n6. Confirmation email/SMS sent via NotificationModule\n\n## 6. TTL (5 min)\n- **Cache**: Business search results, popular categories (TTL 5 min)\n- **Queue**: Background jobs (notifications, email)\n\n## 7. Key Technical Decisions\n\n### 7.1 Why Modular Monolith (NestJS) Instead of Microservices?\n- Team size small, reduces operational complexity\n- Easier to maintain data consistency (single database transactions)\n- Future extraction to microservices is straightforward due to NestJS module boundaries\n\n### 7.2 Why Supabase?\n- Provides real-time capabilities out of the box\n- Managed authentication (OAuth, phone) reduces boilerplate\n- Storage for business images\n- Can be self-hosted if needed\n\n### 7.3 Why PostGIS?\n- Geospatial queries for location-based search\n- Efficient indexing for radius queries\n- Built-in functions for distance calculation, clustering\n\n### 7.4 Optimistic Locking for Slots\n- Each slot has a `version` field\n- On booking, `UPDATE ... WHERE version = :oldVersion`\n- If no rows updated, slot was taken — client retries\n- Redis hold prevents double-booking within 5 minutes\n\n## 8. Security Considerations\n\n- **Authentication**: JWT with short-lived access tokens, refresh token rotation\n- **Rate Limiting**: 5 req/min on auth endpoints, 100 req/min on search\n- **Password Storage**: bcrypt with cost factor 12\n- **Input Validation**: Zod schemas on both client and server\n- **SQL Injection**: Prevented by Prisma parameterized queries\n- **XSS**: React Native handles by default, API sanitizes HTML\n- **CORS**: Strict origin whitelist\n- **Data Privacy**: GDPR compliance, data export, 30-day deletion grace period\n\n## 9.
