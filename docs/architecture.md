  "content": "# Planity Clone — System Architecture\n\n## 1. Overview\nPlanity Clone is a mobile-first booking platform for beauty & wellness businesses. The architecture follows clean, modular monorepo principles using Nx and pnpm, separating mobile client, backend API, and shared libraries. Priorities: P0 booking path, P1 engagement, P2 provider/admin portals.\n\n## 2. Technology Stack\n- **Mobile:** Expo, React Native, TypeScript, Expo Router, TanStack React Query, React Native Reanimated.\n- **Backend:** NestJS, Prisma, PostgreSQL + PostGIS, Redis (cache & BullMQ), Supabase (managed Postgres/Storage).\n- **Tooling:** Nx, pnpm, Docker Compose, GitHub Actions, EAS Build, Jest.\n\n## 3. Monorepo Layout (Nx + pnpm)\nRoot contains workspace config, Docker, CI. Applications live in `apps/`, reusable libraries in `libs/`.\n\n## 4. Service Boundaries\n### 5.1 Mobile App (Expo)\n- **Routing:** Expo Router with route groups for auth, tabs, business detail, booking flow.\n- **Data Layer:** `libs/api-client` exposes typed React Query hooks; mobile features call hooks, no direct fetch.\n- **UI Layer:** `libs/ui` provides theme tokens and components; Reanimated powers bottom sheets, map preview, transitions.\n- **State:** Server state via React Query; local UI state (cart, filters) via React context or lightweight store.\n- **Offline/Guest:** Guest browse allowed; auth gate on booking.\n\n### 5.2 Backend API (NestJS)\nModular by domain. Each module contains controller (HTTP), service (logic), and Prisma repository. Cross-cutting:\n- **Auth Module:** Email/password (bcrypt), phone OTP, social (Google/Apple) → JWT access + Redis refresh tokens.\n- **Business & Category:** CRUD, ownership via Provider Portal.\n- **Search Module:** Uses PostGIS `ST_DWithin` for radius queries; Redis caches hot searches.\n- **Availability Module:** Pure function computing 15-min slots from hours, staff shifts, bookings; timezone aware.\n- **Booking Module:** Orchestrates availability, payment, notification; transactional.\n- **Payment Module:** Stripe/PayPal integration; PCI-compliant; refunds.\n- **Notification Module:** Push (Expo/FCM/APNs), email, SMS; templates; preferences.\n- **Review/Favorite:** Linked to completed bookings.\n- **Jobs Module:** BullMQ queues (reminders, slot cache warming, analytics) with Redis.\n\n### 5.3 Shared Libraries\n- `shared-types`: Single source of truth for API contracts and entities.\n- `ui`: RN components consuming theme; used by mobile only (web later).\n- `api-client`: React Query hooks generated from shared-types.\n\n### 5.4 Infrastructure\n- **PostgreSQL + PostGIS:** Primary datastore; geo indexes on business location.\n- **Supabase:** Hosts Postgres + PostGIS and Storage buckets for images/avatars; Prisma connects via connection string.\n- **Redis:** Cache (search/slots) and BullMQ broker.\n- **Docker Compose:** Local dev spins up api, postgres-postgis, redis.\n- **GitHub Actions:** CI runs Nx affected tests/build; lint; Jest. Mobile workflow triggers EAS Build for preview/prod.\n\n## 6. Key Architectural Decisions\n- **Monorepo:** Nx + pnpm ensures type-safe contracts between mobile and API via `shared-types`.\n- **Prisma + PostGIS:** Use raw SQL for geo queries via Prisma `$queryRaw` or extension; migrations versioned.\n- **Caching:** Search results and computed slots cached in Redis with TTL; invalidated on booking/business update.\n- **Async Work:** BullMQ isolates notification sending and reminders; dead-letter queue monitored.\n- **Security:** JWT stateless auth; refresh tokens in Redis with rotation; OTP via Redis short-lived key.\n\n## 7.
