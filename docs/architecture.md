  "content": "# Planity Clone — System Architecture\n\n## 1. Overview\nPlanity Clone is a mobile-first marketplace for beauty/wellness appointments. The system is built as a pnpm + Nx monorepo containing an Expo React Native mobile client, a NestJS API, and shared libraries. Persistence uses PostgreSQL with PostGIS (via Supabase) and Redis for caching/queues. CI/CD via GitHub Actions, EAS Build for mobile, Docker Compose for local infra.\n\n## 2. Design Principles\n- Clean Architecture: separation of concerns between UI, domain, and data.\n- Single Source of Truth: shared TypeScript types and UI kit in libs.\n- API-first: NestJS exposes versioned REST endpoints consumed by React Query.\n- Async by default: heavy tasks via BullMQ/Redis.\n- Type safety end-to-end.\n\n## 3. Monorepo Structure (Nx + pnpm)\n\nWorkspace root:\n```\nplanity-clone/\n├── apps/\n│   ├── mobile/                # Expo RN app (customer)\n│   ├── api/                   # NestJS backend\n│   ├── web-provider/         # React (Vite) provider portal (P0)\n│   └── web-admin/            # React admin dashboard (P1)\n├── libs/\n│   ├── shared-types/         # TS interfaces (User, Business, Booking, Slot)\n│   ├── ui/                   # Design system (Button, Input, Card, Modal)\n│   ├── config/               # ESLint, TS configs, jest presets\n│   ├── backend/\n│   │   ├── prisma/           # Prisma schema, migrations, seed\n│   │   └── redis/            # Redis/BullMQ factory\n│   └── mobile/\n│       ├── api-client/       # Fetch wrapper, React Query hooks\n│       └── auth/             # Auth context, token storage\n├── tools/\n│   ├── docker/               # Dockerfiles\n│   └── scripts/              # Seed, codegen\n├── docker-compose.yml        # postgres-postgis, redis, api\n├── .github/workflows/        # CI/CD\n├── package.json\n├── pnpm-workspace.yaml\n└── nx.json\n```\n\n(Note: web apps use React + Vite; Supabase hosts Postgres + Storage.)\n\n## 4.
