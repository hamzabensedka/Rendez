We use an Nx monorepo with pnpm to share code and enforce boundaries. PostgreSQL + PostGIS handles geo and relational data; Redis powers caching and BullMQ jobs. Supabase provides auth baseline and storage; EAS Build ships the Expo app.\n\n## 2. Service Boundaries\n- **auth**: user registration, login, token issue/verify, password reset, lockout. Uses Supabase Auth + custom JWT.\n- **search**: business text search, category filter, PostGIS radius queries, map viewport results.\n- **booking**: availability engine, slot computation, appointment create/cancel/reschedule, payment trigger.\n- **business**: CRUD business, services, staff, categories, hours; provider dashboard data.\n- **notification**: templated email/push/SMS via queue; user preferences.\n- **admin**: user/business management, disputes, category CRUD, metrics.\n\nEach NestJS service is a module under `apps/api/src/modules/*` and `apps/worker/src/*`. Cross-service calls use internal HTTP or Redis events.\n\n## 4. Backend (NestJS)\n- **API**: REST + JWT guard. Modules: Auth, User, Business, Category, Search, Booking, Review, Payment, Notification, Admin.\n- **Worker**: BullMQ queues (`notify`, `reminder`, `slot-cleanup`). Listens to Redis.\n- **Prisma**: `schema.prisma` with models User, Business, Category, Service, Staff, Appointment, Review, Favorite, Payment, NotificationPref.\n- **PostGIS**: `ST_DWithin` for map search; indexed `geom` column on Business.\n- **Redis**: cache (search results, availability), queue transport, session blacklist.\n\n## 5. Frontend (Expo / React Native)\n- **mobile**: Expo Router tabs (Home, Search, Map, Bookings, Profile). TanStack React Query for server state. Reanimated for sheets, transitions.\n- **provider-portal**: Expo (web target) with calendar, booking list, business editor.\n- **admin-dashboard**: Web with tables and metrics.\n- Shared `libs/ui` provides Button, Card, MapView wrapper, Sheet.\n\n## 6. User picks slot в†’ POST `/booking` в†’ Booking service locks slot (Redis), creates Appointment, emits `notify` job.\n4. Worker sends confirmation; on cancel, frees slot and notifies provider.\n\n## 7.
