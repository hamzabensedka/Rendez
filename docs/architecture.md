optional GraphQL.\n- Prisma schema with geography column for location queries.\n- BullMQ workers: notificationSender, slotCalculator, reminderSender.\n- Dockerfile for containerization.\n\n## Shared Libs\n- `libs/ui`: design system (buttons, cards, map markers) for RN and web.\n- `libs/types`: TS interfaces (User, Business, Service, Appointment, Review, etc.).\n- `libs/utils`: logger, error helpers, date/geo utils.\n\n## Data Layer\n- PostgreSQL with PostGIS extension.\n- Tables: users, providers, businesses, services, appointments, reviews, favorites, availability.\n- Geography point on businesses for radius search.\n- Prisma models with GIST index.\n- Redis: JWT blacklist, session cache, BullMQ queues.\n\n## Service Boundaries\n| Boundary | Responsibility |\n|----------|----------------|\n| Mobile UI | Presentation, gestures, local state |\n| API Gateway | Auth, validation, routing |\n| Business Domain | CRUD, geo search |\n| Booking Domain | Slot calc, appointment lifecycle, payment trigger |\n| Payment | Stripe processing, webhooks |\n| Notification | Push/email/SMS via BullMQ workers |\n| Provider Portal | Role‑protected UI for managing listings, availability, bookings |\n| Admin Portal (future) | Moderation, analytics, user/provider management |\n\n## DevOps & Deployment\n- Local dev: `docker-compose.yml` runs PostgreSQL (PostGIS), Redis, API.\n- CI: GitHub Actions runs lint, type‑check, tests; on main builds Docker images, pushes registry, triggers EAS Build for mobile, deploys API to staging.\n- Production: API behind NGINX TLS; managed PostGIS & Redis; BullMQ workers scale on queue depth; OTA updates via Expo EAS.\n\n## Security\n- JWT short‑lived access token + refresh token (HTTP‑only).\n- Role‑based guards (Admin, Provider, User).\n- DTO validation with class‑validator; Prisma enforces DB constraints.\n- Rate limiting via NestJS ThrottlerGuard backed by Redis.\n- Secrets via GitHub Secrets / EAS env;
