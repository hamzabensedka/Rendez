  "content":"# Planity Clone - System Architecture\n\n## 1. Overview\nPlanity Clone is a mobile-first marketplace for booking beauty and wellness appointments. The system is built as an Nx monorepo managed with pnpm, containing Expo React Native apps (client, provider), a NestJS API, and a BullMQ worker. Shared TypeScript types and UI live in libs. Data is stored in PostgreSQL with PostGIS (via Supabase) and cached/queued with Redis. CI/CD uses GitHub Actions, Docker Compose for local infra, and EAS Build for mobile releases. Testing with Jest.\n\n## 2. Monorepo Folder Scaffold\n\n```\nplanity-clone/\n├── apps/\n│   ├── mobile/        # Expo + React Native client (Expo Router)\n│   ├── provider/      # Expo provider portal (role-based)\n│   ├── admin/         # React web admin dashboard (Vite or Expo web)\n│   ├── api/           # NestJS REST API\n│   └── worker/        # NestJS microservice consuming BullMQ\n├── libs/\n│   ├── shared/\n│   │   ├── types/     # User, Business, Appointment, etc.\n│   │   ├── ui/        # Button, Card, Input, Modal (design system)\n│   │   └── config/    # Env vars, theme tokens\n│   ├── backend/\n│   │   ├── prisma/    # schema.prisma, migrations, seed\n│   │   ├── core/      # NestJS modules (auth, booking, search...)\n│   │   └── integrations/ # Stripe, Supabase, Geo\n│   ├── mobile/\n│   │   ├── features/  # auth, explore, search, booking, profile\n│   │   ├── api/       # React Query hooks, fetch client\n│   │   ├── animations/# Reanimated helpers\n│   │   └── components/# Mobile-specific UI\n│   └── contracts/     # API DTOs shared with mobile\n├── infra/\n│   ├── docker/        # docker-compose.yml (postgres-postgis, redis)\n│   └── eas/           # eas.json\n├── .github/workflows/ # ci.yml, cd.yml\n├── nx.json\n├── pnpm-workspace.yaml\n└── package.json\n```\n\n## 3. Service Boundaries\n\n### 3.1 Backend (NestJS API + Worker)\n- **AuthModule**: Integrates Supabase Auth for email/password, Google/Apple social, JWT issuance, password reset, logout token invalidation (Redis blacklist).\n- **UsersModule**: Profile CRUD, notification preferences, account deletion.\n- **BusinessesModule**: CRUD business profile, gallery (Supabase Storage), staff management, hours.\n- **CategoriesModule**: Hierarchical taxonomy (Hair > Cut > Women).\n- **ServicesModule**: Service offerings per business, price, duration.\n- **SearchModule**: Text/filter search using PostgreSQL + PostGIS (distance, radius), pagination, sorting. Exposes /search and /map endpoints.\n- **BookingModule**: Orchestrates multi-step booking, calls Availability, creates Appointment, triggers Payment, enqueues Notification job.\n- **AvailabilityModule**: Computes slots from business hours, staff shifts, service duration, buffers; on changes publishes recompute job to Worker.\n- **PaymentsModule**: Stripe integration, tokenized cards, charges, refunds.\n- **ReviewsModule**: Verified reviews (post-appointment), ratings aggregate.\n- **FavoritesModule**: User saved businesses, synced via API.\n- **NotificationsModule**: Push (Expo/Supabase), email, SMS; enqueues BullMQ jobs.\n- **AdminModule**: User/business/category management, disputes, metrics.\n- **ProviderModule**: Business owner portal operations (calendar, analytics).\n- **WorkerService**: BullMQ queues: notifications, slot-recompute, image-optimize. Retries with exponential backoff.\n\n### 3.2 Mobile (Expo Router)\n- **Expo Router screens**: (auth), (tabs), business/[id], booking/..., profile, provider, admin.\n- **Feature modules** in libs/mobile/features: auth, guest-explore, search, map, business-detail, booking, appointments, favorites, profile, provider.\n- **Data layer**: TanStack React Query hooks in libs/mobile/api talk to NestJS API, handle caching, pagination, auth token injection.\n- **Animations**: React Native Reanimated for page transitions, bottom-sheet booking modal, map preview card.\n- **Design system**: libs/shared/ui components themed via shared/config tokens.\n- **Secure storage**: Expo SecureStore for JWT.\n\n### 3.3 Shared\n- **Types**: Single source of truth for entities, DTOs, enums.\n- **UI**: Cross-platform components used by mobile and web admin.\n- **Contracts**: API request/response shapes validated with zod/class-validator.\n\n## 4. Data Architecture\n- **PostgreSQL + PostGIS** (Supabase): tables User, Business (with location geometry(Point,4326)), Service, Category, Appointment, Review, Favorite, Payment. Prisma schema with postgis extension.\n- **Redis**: caches search results, geolocation lookups, JWT blacklist, and serves as BullMQ broker.\n- **Supabase Storage**: business images, avatars, optimized via worker.\n\n## 5. Infrastructure & CI/CD\n- **Docker Compose**: local services postgres-postgis, redis. API and worker run via pnpm nx serve api.\n- **GitHub Actions**: on push, pnpm install, nx affected -t lint test build. Separate workflow for EAS Build (eas build) and Docker image push for API.\n- **EAS Build**: apps/mobile/eas.json configures development/production builds for iOS/Android.\n- **Supabase**: managed Postgres+PostGIS, Auth, Storage in staging/prod.\n\n## 6. Security & Compliance\n- Passwords hashed by Supabase (bcrypt). JWT stored in SecureStore.\n- RBAC: client, provider, admin roles enforced in NestJS guards.\n- PCI: Stripe tokenization, no raw card data in DB.\n- Geolocation permission prompts on mobile.\n\n## 7. Testing\n- **Jest** unit tests for libs and NestJS modules; integration tests with testcontainers (PostGIS, Redis).\n- Mobile component tests with Jest + React Native Testing Library.\n- GitHub Actions runs coverage gate.\n\n## 8. Scalability & Performance\n- Slot computation offloaded to Worker to meet <2s response.\n- Redis caching of hot searches and business details.\n- PostGIS spatial indexes for fast map queries.\n- Nx distributed caching speeds up CI.\n\n## 9.
