Service Boundaries\n\n### 5.1 Auth Service\n- **Responsibility:** User registration, login, token management, password reset, social login, session management.\n- **Endpoints:** POST /auth/register, POST /auth/login, POST /auth/refresh, POST /auth/logout, POST /auth/password-reset, POST /auth/social-login.\n- **Dependencies:** Redis (session store), Supabase (social auth), PostgreSQL (user data).\n\n### 5.2 User Service\n- **Responsibility:** User profile management, preferences, favorites, search history.\n- **Endpoints:** GET /users/me, PATCH /users/me, GET /users/me/favorites, POST /users/me/favorites, DELETE /users/me/favorites.\n- **Dependencies:** PostgreSQL.\n\n### 5.3 Business Service\n- **Responsibility:** Business CRUD, categories, team management, business hours, policies, gallery.\n- **Endpoints:** GET /businesses, GET /businesses/:id, POST /businesses, PATCH /businesses/:id, DELETE /businesses/:id, GET /businesses/:id/services, GET /businesses/:id/team.\n- **Dependencies:** PostgreSQL, PostGIS (location queries), Redis (caching).\n\n### 5.4 Search Service\n- **Responsibility:** Text search, filtering, sorting, autocomplete, trending searches, map-based search.\n- **Endpoints:** GET /search, GET /search/autocomplete, GET /search/trending, GET /search/map.\n- **Dependencies:** PostgreSQL (full-text search with tsvector), PostGIS (geospatial queries), Redis (caching, trending).\n\n### 5.5 Booking Service\n- **Responsibility:** Slot availability, booking creation, cancellation, slot hold, booking history.\n- **Endpoints:** GET /bookings/slots, POST /bookings, GET /bookings, GET /bookings/:id, PATCH /bookings/:id/cancel, POST /bookings/:id/hold.\n- **Dependencies:** PostgreSQL, Redis (slot hold, real-time availability).\n\n### 5.6 Notification Service\n- **Responsibility:** Push notifications, email notifications, in-app notifications.\n- **Endpoints:** POST /notifications/send, GET /notifications, PATCH /notifications/:id/read.\n-
