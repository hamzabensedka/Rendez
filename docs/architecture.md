The mobile client is built with Expo, React Native, TypeScript, Expo Router, TanStack React Query, and React Native Reanimated. The backend consists of NestJS services communicating with a PostgreSQL/PostGIS database, Prisma ORM, Redis (caching & BullMQ job queue), and Supabase for authentication & storage. CI/CD is handled via GitHub Actions, Docker Compose (local dev), and EAS Build (mobile).\n\n---\n\n## High‑Level Architecture\n\n```\n+-------------------+          +-------------------+ PostgreSQL with PostGIS extension for geo‑queries; Prisma as the single source of truth for schema migrations.\n* **Cache & Job Queue** – Redis stores frequently accessed data (e.g., business listings, slot caches) and backs BullMQ workers for background jobs (notifications, slot recomputation, payment retries).\n* **Authentication** – Supabase Auth (email/password + social) used via its JWT; services validate tokens via a shared Supabase client lib.\n* **File Storage** – Supabase Storage for business images, user avatars, and receipts.\n* **DevOps** – Docker Compose spins up all services locally; GitHub Actions runs lint, test, build, and pushes Docker images; EAS Build creates IPA/APK for stores.\n\n---\n\n## Folder Structure (Nx Monorepo)\n\n```\nplanity-clone/\n├──
