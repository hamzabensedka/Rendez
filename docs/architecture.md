  "content": "# Planity Clone — System Architecture\n\n## 1. Introduction\nThis document defines the complete system architecture for Planity Clone, a mobile-first beauty & wellness booking platform. The design follows clean architecture, strict separation of concerns, and scalability principles using the mandated stack.\n\n## 2. Technology Stack\n- **Mobile Client**: Expo, React Native, TypeScript, Expo Router, TanStack React Query, React Native Reanimated\n- **Backend**: NestJS, Prisma, PostgreSQL, PostGIS, Redis, BullMQ\n- **Infra**: Nx, pnpm, Docker Compose, GitHub Actions, Supabase, EAS Build, Jest\n\n## 3. Monorepo Layout (Nx + pnpm)\n```\nplanity-clone/\n├── apps/\n│   ├── mobile/            # Expo React Native app\n│   ├── api/               # NestJS backend\n│   ├── admin/             # Admin dashboard (web)\n│   └── provider/          # Business owner portal (web)\n├── libs/\n│   ├── shared-types/      # TS types, DTOs\n│   ├── ui/                # Design system, RN components\n│   ├── config/            # Shared config, env schemas\n│   └── utils/             # Cross-cutting helpers\n├── services/\n│   ├── auth/              # Auth microservice logic (NestJS module)\n│   ├── booking/           # Booking & slots\n│   ├── search/            # Search & geo\n│   ├── notification/      # Push/email/SMS\n│   └── jobs/              # BullMQ workers\n├── infra/\n│   ├── docker/            # Dockerfiles, Compose\n│   ├── ci/                # GitHub Actions\n│   └── supabase/          # Supabase config\n├── docs/\n└── package.json\n```\n\n##
