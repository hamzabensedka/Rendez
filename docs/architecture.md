Mobile App (`apps/mobile`)\n- **Presentation Layer**: Screens built with Expo Router (file‑based routing).\n- **State Layer**: React Query hooks encapsulate API calls; optimistic updates for booking/favorite actions.\n- **Animation Layer**: Reanimated 2 for shared‑element transitions, animated maps, and gesture‑driven booking calendar.\n- **Device Features**: Expo SDK for camera (profile pic), push notifications (expo-notifications), and biometric auth (optional).\n\n### 2. Web Admin (`apps/web`)\n- Similar to mobile but uses Next.js pages/router.\n- Leverages same UI library (`libs/ui`) for consistency.\n- Uses React Query for data fetching; admin‑specific modules (user management, business verification, analytics).\n\n### 3. API (`apps/api`)\n- **Controllers**: HTTP endpoints (REST) or GraphQL resolvers; validation via class‑validator & DTOs.\n- **Guards**: JWT auth guard (Supabase‑issued tokens), role‑based (USER, PROVIDER, ADMIN).\n- **Services**: Business logic encapsulated; transactional boundaries using Prisma’s `$transaction`.\n- **Repositories**: Thin Prisma wrappers; centralize query building (especially geo queries with PostGIS).\n- **Workers**: BullMQ processors for:\n    - Sending emails/push notifications\n    - Recalculating availability slots nightly\n    - Exporting reports\n- **Middleware**: Request logging, error transformation, rate limiting.\n- **Prisma Schema**: Defines PostgreSQL tables with PostGIS extensions (`businesses` → `location Point`).\n- **Redis**: Stores session cache (optional), BullMQ queues, and short‑lived OTPs.\n\n### 4. Authentication Flow\n1. User signs up/login via Supabase Auth (email/password or social).\n2. Supabase returns JWT (access + refresh).\n3. Mobile/Web store tokens securely (SecureStore / httpOnly cookie).\n4. On each API request, NestJS `AuthGuard` verifies the JWT signature using Supabase’s JWKS.\n5. Role‑based permissions checked in guards/services.\n\n## Availability & Slot Computation\n- Business hours stored as JSON (e.g., `{ \"mon\": [{\"open\":\"09:00\",\"close\":\"18:00\"}] }`).\n- When a user views a business, the API:\n    1.
