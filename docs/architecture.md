Data Flow\n\n### 5.1 Booking Flow\n1. User selects business and service(s) on mobile app\n2. App calls `GET /businesses/:id/availability?date=&staffId=` to fetch available slots\n3. User selects a slot → app calls `POST /bookings` with service IDs, staff ID, datetime\n4. Backend availability module checks slot availability, creates a temporary hold in Redis (10 min TTL)\n5. If payment required, user completes payment → backend confirms booking, releases hold, updates database\n6. Booking confirmation sent via push notification and email\n7. If payment fails or user cancels, hold is released immediately\n\n### 5.2 Search Flow\n1. User enters search query or interacts with map\n2. App calls `GET /search` with query parameters\n3. Backend search module performs full-text search on PostgreSQL (tsvector) combined with PostGIS distance calculation\n4. Results are ranked by relevance (text match) + proximity (distance) + rating\n5. Results are cached in Redis for 5 minutes for identical queries\n6. Recent searches are stored in Redis per user (max 10)\n\n## 6. Key Design Decisions\n\n### 6.1 Monorepo with Nx\n- Single source of truth for shared types, constants, and utilities\n- Efficient task orchestration with caching and dependency graph\n- Consistent tooling across mobile and backend\n\n### 6.2 TanStack React Query for Server State\n- Automatic caching, background refetching, and optimistic updates\n- Reduces boilerplate for API calls and state management\n- Built-in pagination and infinite scroll support\n\n### 6.3 NestJS Modular Architecture\n- Each business domain is a self-contained module with its own controller, service, and DTOs\n- Dependency injection promotes testability and loose coupling\n- Guards and interceptors enforce cross-cutting concerns (auth, logging, validation)\n\n### 6.4 Slot Computation with Redis Locks\n- Slot availability is computed on-the-fly from business hours, staff schedules, and existing bookings\n- Redis is used for temporary slot holds (10-minute TTL) to prevent double-booking during checkout\n- Real-time updates via Supabase Realtime for calendar views\n\n### 6.5 Geospatial Search with PostGIS\n- PostgreSQL with PostGIS extension enables efficient radius-based queries\n- Indexed geometry columns for fast distance calculations\n- Supports complex spatial queries (e.g., within polygon, nearest neighbor)\n\n## 7. Security Considerations\n\n- JWT tokens with short expiry (15 min) and refresh tokens (30 days)\n- Role-based access control enforced at API gateway and module level\n- Input validation using class-validator and NestJS pipes\n- Rate limiting on auth endpoints (Redis-backed)\n- PCI compliance handled by payment gateway (Stripe) – no raw card data stored\n- GDPR compliance: data export, account deletion with 30-day grace period\n\n## 8. Testing Strategy\n\n- **Unit tests:** Jest for individual services and utilities\n- **Integration tests:** Test NestJS modules with real database (testcontainers)\n- **E2E tests:** Detox for mobile app, supertest for API\n- **CI pipeline:** GitHub Actions runs lint, type-check, unit tests, integration tests on every PR\n\n## 9. Deployment\n\n- **Mobile:** EAS Build for iOS and Android builds, distributed via TestFlight and Google Play\n- **Backend:** Docker container deployed to cloud (AWS ECS or Google Cloud Run)\n- **Database:** Managed PostgreSQL (Supabase or AWS RDS)\n- **Redis:** Managed Redis (Upstash or AWS ElastiCache)\n- **CI/CD:** GitHub Actions with separate workflows for staging and production\n\n## 10. Monitoring & Observability\n\n- **Logging:** Structured JSON logging with Winston\n- **Metrics:** Prometheus metrics for API latency, error rates, booking success rate\n- **Tracing:** OpenTelemetry for distributed tracing\n- **Alerting:** PagerDuty for critical failures (booking failures, high error rates)\n\n## 11. Conclusion\n\nThis architecture provides a scalable, maintainable foundation for the Planity Clone. The modular design allows independent development and testing of each business domain, while the monorepo ensures consistency across the mobile and backend codebases.
