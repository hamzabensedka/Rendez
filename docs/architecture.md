Service Boundaries\n- **Auth Module**: Validates Supabase JWT, exposes user context. No business logic.\n- **Businesses / Services / Staff / Categories**: CRUD owned by provider; public read for guest.\n- **Availability**: Pure compute service. Inputs: businessId, serviceId, date, staffId. Uses PostGIS for timezone/locale and Redis cache (key: `slots:{businessId}:{serviceId}:{date}`). Invalidated on booking change.\n- **Bookings**: Orchestrates availability + payments + notifications. Emits events to queue.\n- **Payments**: Stripe integration; webhooks update booking status.\n- **Notifications**: API enqueues jobs; BullMQ workers send push (Firebase), email (Supabase), SMS.\n- **Reviews**: Gated by completed appointment (query join).\n- **Favorites**: User-scoped toggles.\n- **Admin**: Separate role-guarded controllers.\n\n## 7.
