Executive Summary\n\nPlanity Clone is a mobile-first platform connecting customers with beauty and wellness salons for seamless appointment booking. The system is designed as a monorepo with clear separation between the mobile frontend (Expo/React Native) and the backend API (NestJS), supported by PostgreSQL with PostGIS for geospatial queries, Redis for caching and real-time features, and Supabase for authentication and file storage. The architecture prioritizes scalability, maintainability, and developer experience through Nx workspace management, pnpm for package management, and Docker Compose for local development.\n\n## 2. 3. Technology Stack\n\n### 3.1 Frontend (Mobile)\n- **Framework:** Expo SDK 52+ with React Native 0.76+\n- **Language:** TypeScript 5.3+\n- **Navigation:** Expo Router v4 (file-based routing)\n- **State Management:** TanStack React Query v5 for server state, React Context for UI state\n- **Animations:** React Native Reanimated v3 for gesture-driven animations\n- **Maps:** react-native-maps with custom clustering\n- **Payments:** Stripe React Native SDK\n- **Authentication:** Supabase Auth (Google, Apple, Email)\n- **Push Notifications:** Expo Notifications + FCM/APNs\n\n### 3.2 Backend\n- **Framework:** NestJS 10+ with Express adapter\n- **Language:** TypeScript 5.3+\n- **ORM:** Prisma 5+ with PostgreSQL\n- **Database:** PostgreSQL 16 with PostGIS extension\n- **Caching:** Redis 7 (ioredis client)\n- **Authentication:** Supabase Auth + custom JWT validation\n- **File Storage:** Supabase Storage\n- **Payments:** Stripe API (server-side)\n- **Email:** Resend or SendGrid\n- **Background Jobs:** BullMQ with Redis\n\n### 3.3 DevOps & Tooling\n- **Monorepo:** Nx 19+ with pnpm workspaces\n- **Package Manager:** pnpm 9+\n- **Containerization:** Docker Compose for local development\n- **CI/CD:** GitHub Actions\n- **Mobile Builds:** EAS Build\n- **Testing:** Jest (unit/integration), Detox (E2E mobile), Supertest (API E2E)\n- **Code Quality:** ESLint, Prettier, Husky pre-commit hooks\n\n## 4. Service Boundaries & Module Design\n\n### 5.1 Auth Module\n**Responsibility:** User registration, login, social auth, password reset, session management, role assignment.\n**Key Features:**\n- Email/password registration with validation\n- Google and Apple OAuth via Supabase\n- JWT token generation and validation\n- Refresh token rotation (30-day session)\n- Password reset flow with 15-minute expiry\n- Role selection (Customer/Provider) post-registration\n\n**API Endpoints:**\n- `POST /auth/register` - Register new user\n- `POST /auth/login` - Email/password login\n- `POST /auth/refresh` - Refresh access token\n- `POST /auth/forgot-password` - Request password reset\n- `POST /auth/reset-password` - Reset password with token\n- `POST /auth/logout` - Invalidate session\n- `GET /auth/me` - Get current user profile\n\n### 5.2 Users Module\n**Responsibility:** User profile management, preferences, favorites.\n**Key Features:**\n- Profile CRUD operations\n- Favorite salons management\n- User preferences (notifications, location)\n- Profile image upload via Supabase Storage\n\n**API Endpoints:**\n- `GET /users/:id` - Get user profile\n- `PATCH /users/:id` - Update profile\n- `GET /users/:id/favorites` - Get favorite salons\n- `POST /users/:id/favorites/:salonId` - Add favorite\n- `DELETE /users/:id/favorites/:salonId` - Remove favorite\n\n### 5.3 Salons Module\n**Responsibility:** Salon CRUD, business details, operating hours, staff management, gallery.\n**Key Features:**\n- Salon profile management (Provider role)\n- Operating hours with special schedules\n- Staff member management\n- Image gallery with Supabase Storage\n- Amenities and policies\n\n**API Endpoints:**\n- `GET /salons` - List salons (paginated, filtered)\n- `GET /salons/:id` - Get salon detail\n- `POST /salons` - Create salon (Provider)\n- `PATCH /salons/:id` - Update salon\n- `GET /salons/:id/staff` - Get salon staff\n- `POST /salons/:id/staff` - Add staff member\n- `PATCH /salons/:id/staff/:staffId` - Update staff\n- `DELETE /salons/:id/staff/:staffId` - Remove staff\n\n### 5.4 Services Module\n**Responsibility:** Service catalog management, pricing, duration, categorization.\n**Key Features:**\n- Service CRUD per salon\n- Category and subcategory assignment\n- Pricing and duration management\n- Staff-service associations\n\n**API Endpoints:**\n- `GET /salons/:salonId/services` - List salon services\n- `POST /salons/:salonId/services` - Create service\n- `PATCH /services/:id` - Update service\n- `DELETE /services/:id` - Delete service\n\n### 5.5 Categories Module\n**Responsibility:** Service category hierarchy management.\n**Key Features:**\n- Category CRUD (Admin only)\n- Subcategory management\n- Icon and image assignment\n- SEO-friendly slugs\n\n**API Endpoints:**\n- `GET /categories` - List all categories\n- `GET /categories/:slug` - Get category with subcategories\n- `POST /categories` - Create category (Admin)\n- `PATCH /categories/:id` - Update category\n- `DELETE /categories/:id` - Delete category\n\n### 5.6 Bookings Module\n**Responsibility:** Appointment booking, rescheduling, cancellation, availability management.\n**Key Features:**\n- Multi-service booking (max 4 services)\n- Staff selection (optional)\n- Real-time availability checking\n- Concurrency control (pessimistic locking)\n- Booking confirmation with QR code\n- Calendar integration (iCal/Google)\n- Cancellation policy enforcement\n\n**API Endpoints:**\n- `GET /bookings` - List user bookings (upcoming/past)\n- `GET /bookings/:id` - Get booking detail\n- `POST /bookings` - Create booking\n- `POST /bookings/:id/cancel` - Cancel booking\n- `POST /bookings/:id/reschedule` - Reschedule booking\n- `GET /salons/:salonId/availability` - Get available slots\n- `GET /bookings/:id/qr` - Get booking QR code\n\n### 5.7 Search Module\n**Responsibility:** Full-text search, geospatial search, filtering, sorting.\n**Key Features:**\n- Full-text search on salon names, services, categories\n- Geospatial search with PostGIS (radius, bounding box)\n- Multi-facet filtering (category, price, rating, availability)\n- Sort options (relevance, rating, distance, price)\n- Pagination with cursor-based infinite scroll\n- Search suggestions (debounced 300ms)\n- Recent searches (stored locally on device)\n\n**API Endpoints:**\n- `GET /search` - Search salons with query and filters\n- `GET /search/suggestions` - Get autocomplete suggestions\n- `GET /search/nearby` - Get nearby salons (map view)\n\n### 5.8 Reviews Module\n**Responsibility:** Review submission, moderation, rating aggregation.\n**Key Features:**\n- Star rating (1-5) with text and optional photos\n- Review moderation (Admin)\n- Rating aggregation per salon\n- Paginated review listing\n- Review summary with star distribution\n\n**API Endpoints:**\n- `GET /salons/:salonId/reviews` - List reviews\n- `POST /salons/:salonId/reviews` - Create review\n- `PATCH /reviews/:id` - Update review\n- `DELETE /reviews/:id` - Delete review\n- `GET /salons/:salonId/reviews/summary` - Get rating summary\n\n### 5.9 Payments Module\n**Responsibility:** Payment processing, refunds, payout management.\n**Key Features:**\n- Stripe integration (server-side)\n- Payment method collection (card, Apple Pay, Google Pay)\n- Payment capture upon booking\n- Hold amount vs full charge based on salon policy\n- Refund processing for cancellations\n- Webhook handling for Stripe events\n\n**API Endpoints:**\n- `POST /payments/create-intent` - Create payment intent\n- `POST /payments/confirm` - Confirm payment\n- `POST /payments/refund` - Process refund\n- `POST /payments/webhook` - Stripe webhook handler\n\n### 5.10 Notifications Module\n**Responsibility:** Push notifications, email notifications, reminders.\n**Key Features:**\n- Push notification via FCM/APNs\n- Email notifications (booking confirmation, reminders)\n- Booking reminders (24h before, 1h before)\n- Cancellation notifications\n- Marketing notifications (opt-in)\n- Background job processing with BullMQ\n\n**API Endpoints:**\n- `POST /notifications/register-device` - Register device token\n- `GET /notifications/preferences` - Get notification preferences\n- `PATCH /notifications/preferences` - Update preferences\n\n## 6.
