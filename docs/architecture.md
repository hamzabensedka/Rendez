Service Boundaries\n\n### 4.1 Auth Service\n- **Responsibility**: User registration, login, OAuth, JWT management, password reset, session management.\n- **Data**: Users table, refresh tokens, OAuth accounts.\n- **Exposes**: REST endpoints for auth flows, GraphQL for user profile.\n- **Dependencies**: Redis (session store), Supabase Auth (optional integration).\n\n### 4.2 User Service\n- **Responsibility**: User profile CRUD, role management, preferences, device management.\n- **Data**: Users, profiles, roles, devices.\n- **Exposes**: REST endpoints for user management.\n\n### 4.3 Business Service\n- **Responsibility**: Business CRUD, categories, staff management, hours of operation, location management (PostGIS).\n- **Data**: Businesses, categories, staff, schedules, locations.\n- **Exposes**: REST endpoints, geospatial queries.\n\n### 4.4 Service Catalog Service\n- **Responsibility**: Service definitions, categories, pricing, duration, add-ons.\n- **Data**: Services, service categories, add-ons.\n- **Exposes**: REST endpoints.\n\n### 4.5 Appointment Service\n- **Responsibility**: Booking flow, slot management, availability engine, hold mechanism (10-min hold with Redis), real-time validation, cancellation.\n- **Data**: Appointments, slots, holds.\n- **Exposes**: REST endpoints, WebSocket for real-time updates.\n- **Dependencies**: Redis (holds, locks), PostgreSQL (appointments).\n\n### 4.6 Search Service\n- **Responsibility**: Full-text search, geospatial search, auto-complete, filtering, sorting, typo-tolerance.\n- **Data**: Indexed business and service data (Elasticsearch or PostgreSQL full-text).\n- **Exposes**: REST endpoints.\n- **Dependencies**: PostgreSQL (PostGIS), Redis (caching).\n\n### 4.7 Review Service\n- **Responsibility**: Review CRUD, rating aggregation, moderation.\n- **Data**: Reviews, ratings.\n- **Exposes**: REST endpoints.\n\n### 4.8 Payment Service\n- **Responsibility**: Payment processing (Stripe), deposit management, refunds, pay-at-venue.\n- **Data**: Payments, invoices, payment methods.\n- **Exposes**: REST endpoints.\n- **Dependencies**: Stripe API.\n\n### 4.9 Notification Service\n- **Responsibility**: Push notifications, email, SMS (Twilio), in-app notifications.\n- **Data**: Notification templates, logs.\n- **Exposes**: REST endpoints, event-driven via message queue.\n- **Dependencies**: Redis (queue), Firebase Cloud Messaging, Twilio, SendGrid.\n\n### 4.10 Admin Service\n- **Responsibility**: Platform analytics, business verification, dispute resolution, fraud detection.\n- **Data**: Admin logs, analytics.\n- **Exposes**: REST endpoints.\n\n## 5. API Design\n\n### 6.1 RESTful Endpoints\n\n- `POST /api/auth/register`\n- `POST /api/auth/login`\n- `POST /api/auth/refresh`\n- `GET /api/users/me`\n- `PUT /api/users/me`\n- `GET /api/businesses` (with query params for search, filter, sort)\n- `GET /api/businesses/:id`\n- `POST /api/businesses` (provider only)\n- `PUT /api/businesses/:id`\n- `GET /api/businesses/:id/services`\n- `POST /api/businesses/:id/services`\n- `GET /api/businesses/:id/availability?date=&staffId=`\n- `POST /api/appointments`\n- `PUT /api/appointments/:id/cancel`\n- `GET /api/appointments/me` (client)\n- `GET /api/businesses/:id/appointments` (provider)\n- `POST /api/reviews`\n- `GET /api/search?q=&lat=&lng=&category=&priceMin=&priceMax=&rating=`\n\n### 6.2 WebSocket Events\n\n- `appointment.created`\n- `appointment.cancelled`\n- `appointment.updated`\n- `slot.released` (when hold expires)\n\n## 7. Key Architectural Decisions\n\n### 7.1 Monorepo with Nx\n- Shared types and utilities across mobile and backend.\n- Enforces consistent code quality and dependency management.\n- Enables incremental builds and caching.\n\n### 7.2 Modular Backend (NestJS)\n- Each feature is a self-contained module with its own controller, service, and data access.\n- Modules can be developed and tested independently.\n- Dependency injection promotes testability.\n\n### 7.3 Real-time Slot Management with Redis\n- Slot holds are stored in Redis with TTL (10 minutes).\n- Optimistic locking prevents double-booking.\n- WebSocket notifications for slot releases.\n\n### 7.4 Geospatial Queries with PostGIS\n- Location data stored as geometry points.\n- Efficient radius and bounding box queries.\n- Supports clustering for map view.\n\n### 7.5 Caching Strategy\n- Redis caches: business details, service lists, search results (short TTL).\n- TanStack React Query handles client-side caching and background refetching.\n- Stale-while-revalidate pattern for search results.\n\n### 7.6 Authentication Flow\n- JWT access tokens (short-lived, 15 minutes) + refresh tokens (long-lived, 30 days).\n- Refresh token rotation with reuse detection.\n- Concurrent session limit enforced via Redis.\n\n## 8.
