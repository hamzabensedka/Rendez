  "content": "# Planity Clone ‚Äî System Architecture\n\n## 1. Introduction\nPlanity Clone is a mobile-first platform for beauty & wellness bookings. Designed by Jordan (Staff Architect) with emphasis on clean separation of concerns and maintainability, this document defines the monorepo structure, service boundaries, and deployment topology using Expo, React Native, TypeScript, Expo Router, TanStack React Query, React Native Reanimated, NestJS, Prisma, PostgreSQL/PostGIS, Redis, Nx, pnpm, Docker Compose, GitHub Actions, Supabase, EAS Build, and Jest.\n\n## 2. Monorepo Structure (Nx + pnpm)\nWe use Nx to enforce module boundaries and pnpm for fast installs.\n\n```\nplanity-clone/\n‚îú‚îÄ‚îÄ apps/\n‚îÇ   ‚îú‚îÄ‚îÄ mobile/                # Expo React Native app\n‚îÇ   ‚îú‚îÄ‚îÄ api/                   # NestJS backend\n‚îÇ   ‚îî‚îÄ‚îÄ web/                   # Provider & Admin portals (React SPA)\n‚îú‚îÄ‚îÄ libs/\n‚îÇ   ‚îú‚îÄ‚îÄ shared/\n‚îÇ   ‚îÇ   ‚îú‚îÄ‚îÄ types/             # Shared TS interfaces (User, Business, etc.)\n‚îÇ   ‚îÇ   ‚îú‚îÄ‚îÄ ui/                # Cross-platform UI components (Button, Card)\n‚îÇ   ‚îÇ   ‚îú‚îÄ‚îÄ config/            # Env schemas, theme tokens\n‚îÇ   ‚îÇ   ‚îî‚îÄ‚îÄ utils/             # Formatters, validators\n‚îÇ   ‚îú‚îÄ‚îÄ backend/\n‚îÇ   ‚îÇ   ‚îú‚îÄ‚îÄ prisma/            # Prisma schema, migrations, seed\n‚îÇ   ‚îÇ   ‚îú‚îÄ‚îÄ redis/             # Redis client, BullMQ factories\n‚îÇ   ‚îÇ   ‚îî‚îÄ‚îÄ common/            # Guards, decorators, base services\n‚îÇ   ‚îî‚îÄ‚îÄ mobile/\n‚îÇ       ‚îú‚îÄ‚îÄ api-client/        # React Query hooks, axios instance\n‚îÇ       ‚îî‚îÄ‚îÄ designsystem/      # Reanimated-based components\n‚îú‚îÄ‚îÄ tools/\n‚îÇ   ‚îú‚îÄ‚îÄ docker/                # Dockerfiles, compose\n‚îÇ   ‚îî‚îÄ‚îÄ github/                # Workflow definitions\n‚îú‚îÄ‚îÄ docs/\n‚îÇ   ‚îú‚îÄ‚îÄ product.md\n‚îÇ   ‚îî‚îÄ‚îÄ architecture.md\n‚îú‚îÄ‚îÄ package.json\n‚îú‚îÄ‚îÄ pnpm-workspace.yaml\n‚îú‚îÄ‚îÄ nx.json\n‚îî‚îÄ‚îÄ tsconfig.base.json\n```\n\n## 3. Applications & Boundaries\n\n### 3.1 Mobile App (`apps/mobile`)\nExpo + React Native with Expo Router. Feature-first organization:\n\n```\napps/mobile/\n‚îú‚îÄ‚îÄ app/                       # File-based routes (Expo Router)\n‚îÇ   ‚îú‚îÄ‚îÄ (auth)/                # login, signup\n‚îÇ   ‚îú‚îÄ‚îÄ (tabs)/                # home, search, bookings, profile\n‚îÇ   ‚îú‚îÄ‚îÄ business/[id].tsx\n‚îÇ   ‚îú‚îÄ‚îÄ booking/...\n‚îÇ   ‚îî‚îÄ‚îÄ _layout.tsx\n‚îú‚îÄ‚îÄ components/                # Presentational pieces\n‚îú‚îÄ‚îÄ features/\n‚îÇ   ‚îú‚îÄ‚îÄ auth/\n‚îÇ   ‚îú‚îÄ‚îÄ explore/\n‚îÇ   ‚îú‚îÄ‚îÄ search/\n‚îÇ   ‚îú‚îÄ‚îÄ map/                   # Reanimated bottom sheet\n‚îÇ   ‚îú‚îÄ‚îÄ booking/\n‚îÇ   ‚îú‚îÄ‚îÄ appointments/\n‚îÇ   ‚îú‚îÄ‚îÄ profile/\n‚îÇ   ‚îî‚îÄ‚îÄ favorites/\n‚îú‚îÄ‚îÄ lib/                       # React Query client, secure storage\n‚îú‚îÄ‚îÄ assets/\n‚îî‚îÄ‚îÄ jest.config.ts\n```\n\nState: Server state via TanStack React Query; local UI state via React context + Reanimated for animations. No Redux.\n\n### 3.2 API Service (`apps/api`)\nNestJS modular monolith with clear module boundaries. Each module owns its domain.\n\n```\napps/api/\n‚îú‚îÄ‚îÄ src/\n‚îÇ   ‚îú‚îÄ‚îÄ main.ts\n‚îÇ   ‚îú‚îÄ‚îÄ app.module.ts\n‚îÇ   ‚îú‚îÄ‚îÄ modules/\n‚îÇ   ‚îÇ   ‚îú‚îÄ‚îÄ auth/             # JWT, bcrypt, social, logout invalidation\n‚îÇ   ‚îÇ   ‚îú‚îÄ‚îÄ users/            # Profile, payment methods\n‚îÇ   ‚îÇ   ‚îú‚îÄ‚îÄ businesses/       # CRUD, photos (Supabase Storage)\n‚îÇ   ‚îÇ   ‚îú‚îÄ‚îÄ categories/       # Taxonomy seed\n‚îÇ   ‚îÇ   ‚îú‚îÄ‚îÄ search/           # Text + PostGIS geo queries\n‚îÇ   ‚îÇ   ‚îú‚îÄ‚îÄ availability/     # Slot computation engine\n‚îÇ   ‚îÇ   ‚îú‚îÄ‚îÄ bookings/         # Create, reschedule, cancel\n‚îÇ   ‚îÇ   ‚îú‚îÄ‚îÄ appointments/     # Management, status\n‚îÇ   ‚îÇ   ‚îú‚îÄ‚îÄ payments/         # Stripe integration\n‚îÇ   ‚îÇ   ‚îú‚îÄ‚îÄ notifications/    # Push (FCM/APN) + email\n‚îÇ   ‚îÇ   ‚îú‚îÄ‚îÄ reviews/          # Ratings\n‚îÇ   ‚îÇ   ‚îú‚îÄ‚îÄ favorites/        # Bookmarks\n‚îÇ   ‚îÇ   ‚îú‚îÄ‚îÄ admin/            # Super admin CRUD\n‚îÇ   ‚îÇ   ‚îú‚îÄ‚îÄ provider/         # Business owner portal API\n‚îÇ   ‚îÇ   ‚îî‚îÄ‚îÄ jobs/             # BullMQ processors\n‚îÇ   ‚îú‚îÄ‚îÄ common/               # DTOs, guards, interceptors\n‚îÇ   ‚îî‚îÄ‚îÄ config/               # Env validation\n```\n\nBoundaries: Modules communicate only via public services. Cross-module needs use `@Inject()` of exported providers. Database access strictly through PrismaService. Redis used for token blacklist, caching search results, and BullMQ queues.\n\n### 3.3 Web Portals (`apps/web`)\nReact SPA (Vite) for Provider & Admin dashboards, reusing `libs/shared/ui` and `libs/shared/types`. Talks to same NestJS API with role-guarded routes.\n\n## 4. Data Layer\n- **PostgreSQL + PostGIS**
