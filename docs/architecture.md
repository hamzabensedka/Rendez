Service Boundaries\n\n### 4.1 Mobile App (Frontend)\n- **Responsibility**: User interface, navigation, state management, offline caching, animations.\n- **Key Decisions**:\n  - Use Expo Router for file-based routing, which simplifies navigation and deep linking.\n  - TanStack React Query manages server state, caching, and optimistic updates (e.g., favorites).\n  - React Native Reanimated for smooth map transitions and gesture-based interactions.\n  - Supabase for authentication (optional, can be replaced by custom JWT).\n- **Data Flow**: Components use custom hooks (e.g., `useBusinesses`) that wrap React Query. Mutations invalidate relevant queries on success.\n\n### 4.2 API Gateway (NestJS)\n- **Responsibility**: Request validation, authentication, authorization, rate limiting, and routing to internal services.\n- **Key Decisions**:\n  - NestJS modules align with domain boundaries (auth, users, businesses, bookings, etc.).\n  - Global guards for JWT validation and role-based access.\n  - Prisma as the ORM for type-safe database access.\n  - Redis for caching (e.g., business search results, session tokens) and job queues (email notifications).\n\n### 4.3 Database Layer\n- **PostgreSQL** with **PostGIS** extension for geospatial queries (e.g., find businesses within radius).\n- **Prisma** schema defines models: User, Business, Service, Staff, Booking, Review, Favorite, Category, etc.\n- **Redis** for:\n  - Session store (refresh tokens).\n  - Rate limiting counters.\n  - Caching of frequent queries (e.g., popular categories).\n  - Pub/Sub for real-time notifications (e.g., booking confirmation).\n\n### 4.4 External Integrations\n- **Supabase**: Provides authentication (optional), file storage for business photos, and real-time subscriptions.\n- **Map Services**: Google Maps or Mapbox for map rendering and geocoding.\n- **Push Notifications**: Expo Push Notifications or Firebase Cloud Messaging.\n\n## 5. Key Architectural Decisions\n\n### 6.1 Monorepo with Nx\n- **Why**: Shared types and utilities between mobile and API reduce duplication. More efficient for large datasets.\n- **How**: API returns `cursor` field in responses. Mobile app passes it as query parameter.\n\n### 6.3 Optimistic Updates for Favorites\n- **Why**: Instant UI feedback improves user experience.\n- **How**: React Query's `onMutate` updates the cache immediately. Other users see the slot as unavailable. On booking confirmation, key is deleted.\n\n### 6.5 Geospatial Queries with PostGIS\n- **Why**: Efficient radius-based search.\n- **How**: Business locations stored as `geography(Point, 4326)`.
