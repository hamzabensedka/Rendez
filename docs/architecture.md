  "content": "# Planity Clone — System Architecture\n\n## 1. Overview\nPlanity Clone is a mobile-first beauty & wellness booking marketplace. The system is built as an Nx monorepo managed by pnpm, containing an Expo React Native mobile app and a NestJS backend, plus shared libraries. Infrastructure uses PostgreSQL with PostGIS, Redis, Supabase, and Docker Compose. CI/CD via GitHub Actions and EAS Build.\n\n## 2. Technology Mapping\n- Expo + React Native + TypeScript: mobile client.\n- Expo Router: file-based navigation and layouts.\n- TanStack React Query: server-state caching, background sync.\n- React Native Reanimated: animations (transitions, map cards, skeletons).\n- NestJS: API gateway and business logic.\n- Prisma: DB access layer for PostgreSQL/PostGIS.\n- PostgreSQL + PostGIS: relational data and geo queries.\n- Redis: caching, distributed locks, BullMQ job queues.\n- Supabase: Auth (email/phone/OAuth), Storage (images), Realtime (provider calendar).\n- Nx + pnpm: monorepo tooling.\n- Docker Compose: local Postgres/PostGIS + Redis.\n- GitHub Actions: CI pipelines.\n- EAS Build: mobile build & submit.\n- Jest: unit/integration tests.\n\n## 3. Repository & Folder Scaffold\nBelow is the Nx pnpm workspace layout.\n\n```\nplanity-clone/\n├── apps/\n│   ├── mobile/                # Expo Router app\n│   │   ├── app/\n│   │   │   ├── (guest)/       # Home, explore, business detail (no auth)\n│   │   │   ├── (auth)/        # Login, register, forgot-password\n│   │   │   ├── (tabs)/        # Authenticated tabs: Bookings, Profile, Favorites\n│   │   │   └── _layout.tsx\n│   │   ├── src/\n│   │   │   ├── components/    # UI components consuming libs/ui\n│   │   │   ├── hooks/         # Custom React Query hooks\n│   │   │   ├── animations/    # Reanimated builders\n│   │   │   └── lib/           # API client, env config\n│   │   ├── assets/\n│   │   ├── app.json\n│   │   ├── eas.json\n│   │   └── package.json\n│   ├── api/                   # NestJS service\n│   │   ├── src/\n│   │   │   ├── modules/\n│   │   │   │   ├── auth/\n│   │   │   │   ├── users/\n│   │   │   │   ├── businesses/\n│   │   │   │   ├── search/\n│   │   │   │   ├── availability/\n│
