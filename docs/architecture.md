Critical Flows\n\n### 8.1 Slot Availability Computation\n\n```\n1. Client requests availability for service + date range\n2. Service fetches: business hours, staff schedules, existing appointments\n3. Algorithm:\n   a. Generate candidate slots from business hours\n   b. Subtract: existing appointments, staff unavailability\n   c. Apply: service duration, buffer time, concurrent limits\n4. Check Redis for active holds (exclude held slots)\n5. Cache result in Redis (2 min TTL)\n6. Return to client\n```\n\n### 8.2 Booking with Distributed Lock\n\n```\n1. Client selects slot → POST /bookings/hold\n2. Server:\n   a. SET hold:{serviceId}:{startTime} {sessionId} NX EX 300\n   b. If lock fails → 409 Conflict (slot unavailable)\n   c. Return holdId with expiry timestamp\n3.
