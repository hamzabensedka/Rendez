Service Boundaries\n- **Mobile App**: Thin view layer. Uses TanStack React Query for server state, Reanimated for transitions. No business logic; calls API via api-client. Expo Router handles navigation.\n- **API Gateway (NestJS)**: Single backend for all clients. Modular by domain. Validates JWT from Supabase, enforces authorization. Exposes REST (OpenAPI) endpoints.\n- **Database (PostgreSQL + PostGIS)**: Source of truth. PostGIS enables geo queries for map/search. Prisma migrates schema.\n- **Redis**: Caches hot reads (business lists, categories) and backs BullMQ queues for notifications, analytics, pending-booking expiry.\n- **Supabase**: Manages authentication (email, phone OTP, Google/Apple) and file storage (images, avatars, review photos). Issues JWTs; API verifies via middleware.\n- **Background Workers**: NestJS jobs module consumes BullMQ queues; runs outside request lifecycle.\n- **Web Portals**: Provider and Admin apps are React apps that consume the same API with role-scoped tokens.\n\n## 5. Key Domain Modules & Responsibilities\n1. **Auth**: Supabase handles sign-up/login/OTP/social. API auth module syncs user record, stores profile, issues session mapping.\n2. **Businesses & Categories**: CRUD via provider portal; public read for mobile. Categories hierarchical (self-relation).\n3. **Search & Discovery**: search module uses PostGIS ST_DWithin for radius, full-text tsvector for text. Results paginated; Redis caches filter sets.\n4. **Availability Engine**: Pure TS service computing 15-min slots from business hours, staff shifts, service duration, existing bookings, timezone. Returns available slots; no persistence of slots.\n5. **Booking & Appointments**: Orchestrates availability check, payment intent, confirmation. Uses Redis lock to prevent double-booking. Emits events to notifications and jobs.\n6. **Payments**: Integrates Stripe/Adyen. Tokenized cards; deposits/full prepay. On success, finalize booking; on failure, rollback.\n7. **Reviews**: Only after verified appointment; updates business rating (DB trigger or service). Moderation queue.\n8. **Favorites**: Simple join table; synced via React Query.\n9. **Notifications**: Producers push jobs to Redis; workers send FCM/APNs/email/SMS.\n10. **Provider/Admin**: Role-guarded endpoints for managing catalog, staff, analytics.\n\n## 6.
