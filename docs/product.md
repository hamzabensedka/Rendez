Product Overview\nPlanity Clone is a mobile-first beauty & wellness booking platform connecting customers with salons, spas, barbers, and independent professionals. The platform enables seamless discovery, real-time booking, appointment management, and business operations.\n\n**Target Users:** End customers seeking beauty/wellness services, Service providers/business owners, Platform administrators\n**Platforms:** iOS, Android, Responsive Web\n**Core Value Proposition:** Real-time availability discovery and instant booking confirmation\n\n---\n\n## 2. Shared Types & Design System\n\n### 2.1 Core Data Models\n```typescript\n// User\ninterface User {\n  id: string;\n  email: string;\n  phone: string;\n  firstName: string;\n  lastName: string;\n  role: 'customer' | 'provider' | 'admin';\n  avatar?: string;\n  createdAt: Date;\n  updatedAt: Date;\n}\n\n// Business\ninterface Business {\n  id: string;\n  ownerId: string;\n  name: string;\n  description: string;\n  category: ServiceCategory;\n  address: Address;\n  location: GeoPoint;\n  photos: string[];\n  rating: number;\n  reviewCount: number;\n  operatingHours: OperatingHours;\n  isVerified: boolean;\n  status: 'active' | 'inactive' | 'pending';\n}\n\n// Service\ninterface Service {\n  id: string;\n  businessId: string;\n  name: string;\n  description: string;\n  duration: number; // minutes\n  price: number;\n  currency: string;\n  category: ServiceCategory;\n  isActive: boolean;\n}\n\n// Appointment\ninterface Appointment {\n  id: string;\n  customerId: string;\n  businessId: string;\n  serviceId: string;\n  staffId?: string;\n  startTime: Date;\n  endTime: Date;\n  status: 'pending' | 'confirmed' | 'in_progress' | 'completed' | 'cancelled' | 'no_show';\n  price: number;\n  paymentStatus: 'unpaid' | 'paid' | 'refunded';\n  notes?: string;\n}\n\n// Review\ninterface Review {\n  id: string;\n  customerId: string;\n  businessId: string;\n  appointmentId: string;\n  rating: number; // 1-5\n  comment: string;\n  photos?: string[];\n  createdAt: Date;\n}\n```\n\n### 2.2 Design System\n- **Colors:** Primary (#FF6B6B coral), Secondary (#4ECDC4 teal), Neutral grays\n- **Typography:** Inter font family, scale from 12px-32px\n- **Spacing:** 4px grid system (4,8,12,16,20,24,32,48,64)\n- **Components:** Button (primary, secondary, outline, ghost), Card, Input, Modal, BottomSheet, Toast, Badge, Avatar, RatingStars, SkeletonLoader\n- **Icons:** Lucide Icons library\n- **Accessibility:** WCAG 2.1 AA compliance, minimum 4.5:1 contrast ratio\n\n---\n\n## 3. Feature Specifications\n\n### 3.1 User Authentication\n**Priority:** P0 (Critical)\n**Description:** Secure multi-method authentication system supporting email/password, phone OTP, and social login.\n\n**Functional Requirements:**\n- Email/password registration with validation\n- Phone number registration with SMS OTP verification\n- Social login (Google, Apple, Facebook)\n- Password reset flow via email\n- Session management with JWT tokens (access + refresh)\n- Biometric authentication option (FaceID/TouchID)\n- Role-based access control (customer, provider, admin)\n\n**Acceptance Criteria:**\n- [ ] User can register with email within 30 seconds\n- [ ] OTP is delivered within 10 seconds\n- [ ] Invalid credentials show clear error messages\n- [ ] Session persists across app restarts (30 days)\n- [ ] Refresh token rotation implemented\n- [ ] Maximum 5 failed login attempts before temporary lockout (15 min)\n- [ ] All auth endpoints respond within 500ms\n\n**Technical Notes:**\n- Use bcrypt for password hashing\n- JWT access token: 15 min expiry, refresh token: 7 days\n- Store refresh tokens in secure HTTP-only cookies (web) or Keychain/Keystore (mobile)\n\n---\n\n### 3.2 Guest Browse & Explore\n**Priority:** P1 (High)\n**Description:** Allow unauthenticated users to browse businesses and services with limited functionality to encourage signup.\n\n**Functional Requirements:**\n- Browse featured businesses on home screen\n- Search businesses by name, category, location\n- View business profiles with limited photos\n- See service listings and prices\n- Attempting to book triggers registration prompt\n- View up to 10 businesses before signup prompt\n\n**Acceptance Criteria:**\n- [ ] Guest can browse without login for 5 minutes\n- [ ] Booking CTA shows registration modal with clear value proposition\n- [ ] Guest state persists across app sessions (viewed businesses)\n- [ ] Location permission requested only when needed for search\n- [ ] Clear visual indicator showing guest mode\n\n---\n\n### 3.3 Business Search & Discovery\n**Priority:** P0 (Critical)\n**Description:** Powerful search engine enabling users to find businesses by multiple criteria with instant results.\n\n**Functional Requirements:**\n- Full-text search across business name, description, services\n- Filters: category, price range, rating, distance, availability\n- Sort: relevance, rating, distance, price (low/high), popularity\n- Recent searches history\n- Search suggestions/autocomplete\n- Search results with infinite scroll pagination\n- Save search preferences\n\n**Acceptance Criteria:**\n- [ ] Search results return within 300ms\n- [ ] Autocomplete suggestions appear within 200ms\n- [ ] Filters can be combined without page reload\n- [ ] Empty state with helpful suggestions when no results\n- [ ] Search works offline for recent results (cached)\n- [ ] Typo-tolerant search (fuzzy matching)\n- [ ] Results update when map viewport changes\n\n**Technical Notes:**\n- Implement Elasticsearch or Meilisearch for full-text search\n- Use debouncing (300ms) for search input\n- Cache recent searches locally\n\n---\n\n### 3.4 Map-based Search\n**Priority:** P1 (High)\n**Description:** Interactive map view for geographic business discovery with clustering and real-time filtering.\n\n**Functional Requirements:**\n- Map view with business pins/clusters\n- Cluster expansion on zoom\n- Pin shows business preview card on tap\n- Search this area button on map movement\n- Toggle between list view and map view\n- Current location detection\n- Radius filter (1km, 5km, 10km, 25km)\n- Directions integration (Apple Maps/Google Maps)\n\n**Acceptance Criteria:**\n- [ ] Map loads within 2 seconds\n- [ ] Clusters update within 500ms of zoom change\n- [ ] Business pins show rating and category icon\n- [ ] Location permission gracefully handled with fallback\n- [ ] Map view and list view stay synchronized\n- [ ] At least 50 pins rendered without performance degradation\n\n---\n\n### 3.5 Business Detail View\n**Priority:** P0 (Critical)\n**Description:** Comprehensive business profile page with all information needed to make booking decisions.\n\n**Functional Requirements:**\n- Photo gallery with full-screen viewer\n- Business info: name, rating, review count, address, description\n- Services list with prices and durations\n- Staff/professional profiles if applicable\n- Reviews section with sorting and filtering\n- Operating hours with real-time open/closed status\n- Amenities and features tags\n- Share business profile\n- Call business button\n- Favorite/unfavorite toggle\n\n**Acceptance Criteria:**\n- [ ] Page loads within 2 seconds\n- [ ] Photo gallery supports swipe gestures\n- [ ] Open/closed status updates in real-time based on timezone\n- [ ] Reviews lazy-load with infinite scroll\n- [ ] Service selection highlights available time slots\n- [ ] All CTAs have clear visual hierarchy\n- [ ] Page works offline with cached data\n\n---\n\n### 3.6 Service Categories\n**Priority:** P1 (High)\n**Description:** Hierarchical category system for organizing beauty and wellness services.\n\n**Functional Requirements:**\n- Top-level categories: Hair, Nails, Skin, Massage, Barber, Makeup, Wellness, Spa\n- Sub-categories (e.g., Hair > Cut, Color, Styling, Treatment)\n- Category icons and images\n- Browse by category with featured businesses\n- Category-specific filters\n- Trending categories based on location/season\n\n**Acceptance Criteria:**\n- [ ] Category hierarchy supports 3 levels maximum\n- [ ] Category page shows relevant businesses within 1 second\n- [ ] Categories are localizable\n- [ ] Admin can add/modify categories without deployment\n- [ ] Category images follow consistent aspect ratio (16:9)\n\n---\n\n### 3.7 Booking Flow\n**Priority:** P0 (Critical)\n**Description:** Streamlined booking experience from service selection to confirmation with minimal friction.\n\n**Functional Requirements:**\n- Step 1: Select service(s) from business profile\n- Step 2: Choose staff member (optional, \"any available\" default)\n- Step 3: Select date from calendar with availability indicators\n- Step 4: Select time slot from available options\n- Step 5: Review booking summary (service, date, time, price, duration)\n- Step 6: Add notes/special requests\n- Step 7: Confirm and pay (or pay later if supported)\n- Progress indicator showing current step\n- Back navigation preserves selections\n\n**Acceptance Criteria:**\n- [ ] Complete booking flow in under 2 minutes\n- [ ] Available slots update in real-time\n- [ ] Double-booking prevention with server-side validation\n- [ ] Clear error recovery if slot becomes unavailable\n- [ ] Booking confirmation screen with calendar add option\n- [ ] Guest users prompted to register at confirmation step\n- [ ] Booking flow works with intermittent connectivity (optimistic UI)\n\n---\n\n### 3.8 Appointment Management\n**Priority:** P0 (Critical)\n**Description:** Centralized appointment hub for customers to view, manage, and track all bookings.\n\n**Functional Requirements:**\n- Upcoming appointments list with countdown\n- Past appointments history\n- Appointment details: business, service, date/time, status, price\n- Cancel appointment (with policy enforcement)\n- Reschedule appointment (re-enter booking flow)\n- Add to calendar (Google, Apple, Outlook)\n- Reminder notifications configuration\n- Rebook same service shortcut\n- Appointment status tracking (confirmed, in-progress, completed)\n\n**Acceptance Criteria:**\n- [ ] Cancellation respects business cancellation policy (e.g., 24h notice)\n- [ ] Reschedule preserves original service selections\n- [ ] Calendar sync works for all major providers\n- [ ] Appointment list loads within 500ms\n- [ ] Status updates reflect in real-time\n- [ ] Clear visual distinction between upcoming and past appointments\n\n---\n\n### 3.9 Favorites\n**Priority:** P2 (Medium)\n**Description:** Allow users to save favorite businesses for quick access and personalized recommendations.\n\n**Functional Requirements:**\n- Favorite/unfavorite from business card and detail view\n- Favorites list with sorting (recent, rating, distance)\n- Quick book from favorites\n- Favorites sync across devices (authenticated users)\n- Favorite-based recommendations\n- Collections/folders for organizing favorites (v2)\n\n**Acceptance Criteria:**\n- [ ] Favorite toggle responds within 200ms\n- [ ] Favorites persist across sessions\n- [ ] Favorites list shows availability status\n- [ ] Empty state with discovery suggestions\n- [ ] Maximum 500 favorites per user\n\n---\n\n### 3.10 User Profile\n**Priority:** P1 (High)\n**Description:** User account management and personalization hub.\n\n**Functional Requirements:**\n- Profile photo upload with crop tool\n- Personal information data export option\n- Login history and device management\n\n**Acceptance Criteria:**\n- [ ] Profile updates save within 1 second\n- [ ] Photo upload supports JPEG, PNG, WebP (max 5MB)\n- [ ] Account Availability & Slot Computation\n**Priority:** P0 (Critical)\n**Description:** Real-time availability engine computing bookable time slots based on business hours, staff schedules, existing Recurring availability patterns\n\n**Acceptance Criteria:**\n- [ ] Slot computation completes within 200ms\n- [ ] Slot hold expires and releases after 5 minutes\n- [ ] Concurrent booking attempts services)\n- [ ] 99.9% accuracy in availability display\n\n**Technical Notes:**\n- Use Redis for slot locking and real-time availability cache\n- Compute availability in UTC, display in user's timezone\n- Implement optimistic locking for appointment creation\n\n---\n\n### 3.12 Reviews & Ratings\n**Priority:** P1 (High)\n**Description:** Trust-building review system allowing customers to rate and review businesses after appointments.\n\n**Functional Requirements:**\n- Star rating (1-5) with half-star precision\n- Written review with minimum 10 characters\n- Photo upload with reviews popular services, occupancy rates\n\n7. **Email Campaigns**\n   - Schedule: As triggered by admin\n   - Batch: 1000 recipients per job\n   - Throttle: 50 emails/second\n\n**Acceptance Criteria:**\n- [ ] Job processing delay < 30 seconds under normal load\n- [ ] Failed jobs retry with exponential backoff\n- [ ] Dead letter queue for jobs failing after all retries\n- [ ] Job dashboard showing queue Notes:**\n- Use BullMQ with Redis backend\n- Implement job priorities (critical: reminders, normal: analytics, low: cleanup)\n- Monitor queue health with Bull Board UI\n\n---\n\n## 4.
